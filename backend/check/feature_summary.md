# Backend Feature Summary

이 문서는 백엔드 코드의 컨트롤러를 분석하여 현재까지 구현된 기능 목록을 요약합니다.

## 1. 인증 (Authentication) - `AuthController`

- **Google OAuth2 기반 인증:**
    - Google ID Token 또는 Access Token을 이용한 로그인 및 회원가입 처리.
    - 신규 사용자일 경우, 기본적인 사용자 정보를 생성하고 `profileCompleted` 상태를 `false`로 설정.
    - 기존 사용자일 경우, 로그인 처리 후 JWT 토큰 발급.
- **로그아웃:**
    - 현재 세션을 종료하는 기능을 제공 (클라이언트 측 토큰 삭제로 처리).
- **디버그용 API:**
    - 모든 사용자의 프로필 완성 상태(`profileCompleted`)를 `false`로 초기화하는 기능.

## 2. 사용자 및 프로필 (User & Profile) - `UserController`, `MeController`, `RoleController`

- **온보딩 (신규 사용자 정보 입력):**
    - 첫 로그인 후, 사용자의 상세 정보(닉네임, 역할, 학적 정보 등)를 입력받아 프로필을 완성.
- **프로필 관리:**
    - 닉네임 중복 확인 및 추천 기능.
    - 사용자 프로필 정보(닉네임, 자기소개, 프로필 이미지 등) 업데이트.
- **내 정보 조회 (`/api/me`):**
    - 현재 로그인된 사용자의 전체 프로필 정보를 반환.
- **사용자 검색:**
    - 이름, 이메일 등으로 사용자를 검색하는 기능 (역할 필터링 포함).
- **역할 신청:**
    - 사용자가 특정 역할(예: `PROFESSOR`)을 신청하고, 승인 대기 상태로 전환하는 기능.
- **내 활동 조회:**
    - 내가 보낸 그룹 가입 신청 목록 및 상태 조회.
    - 내가 보낸 하위 그룹 생성 신청 목록 및 상태 조회.

## 3. 그룹 관리 (Group Management) - `GroupController`

- **그룹 CRUD:**
    - 그룹 생성, 조회, 정보 수정, 삭제 기능.
- **그룹 탐색 및 검색:**
    - 다양한 조건(모집 상태, 공개 여부, 그룹 유형, 소속, 태그 등)을 조합하여 그룹을 검색하는 기능.
    - 전체 그룹 목록 및 페이징 처리된 목록 조회.
    - 온보딩 시 학과/동아리 선택 UI를 위한 계층 구조 데이터 제공.
- **멤버십 관리:**
    - 그룹 가입 신청 및 탈퇴.
    - 특정 그룹의 멤버 목록 조회.
    - 멤버의 역할(예: 관리자, 멤버) 변경 및 강제 탈퇴.
    - 현재 사용자의 특정 그룹 멤버십 여부 확인 (단일/일괄).
- **그룹 내 역할(Role) 관리:**
    - 그룹 내에서 사용될 역할(예: 회장, 총무)에 대한 CRUD 기능.
- **가입 신청 처리:**
    - 그룹 관리자가 가입 신청을 조회하고 승인/거절하는 기능.
- **하위 그룹 관리:**
    - 하위 그룹 생성을 신청하고, 상위 그룹 관리자가 이를 승인/거절하는 기능.
    - 특정 그룹에 속한 하위 그룹 목록 조회.
- **지도교수 관리:**
    - 그룹에 지도교수를 배정하거나 해제하는 기능.
- **소유권 이전:**
    - 그룹 소유자(리더)를 다른 멤버에게 위임하는 기능.

## 4. 콘텐츠 및 워크스페이스 (Content & Workspace) - `ContentController`

- **워크스페이스:**
    - 그룹 내에 여러 개의 워크스페이스를 생성, 조회, 수정, 삭제하는 기능 (현재는 그룹당 단일 워크스페이스 개념과 호환).
- **채널:**
    - 워크스페이스 내에 주제별 채널을 생성, 조회, 수정, 삭제하는 기능.
- **게시글 (Post):**
    - 채널 내에 게시글을 작성, 조회, 수정, 삭제하는 기능.
- **댓글 (Comment):**
    - 게시글에 댓글을 작성, 조회, 수정, 삭제하는 기능.
- **권한 관리:**
    - 현재 사용자가 특정 채널에 대해 어떤 권한(읽기, 쓰기 등)을 가지고 있는지 확인하는 기능.

## 5. 관리자 (Admin) - `AdminController`

- **전체 가입 신청 관리:**
    - 시스템 전체의 그룹 가입 신청을 상태별로 조회하고 관리(승인/거절)하는 기능.
- **전체 그룹 생성 신청 관리:**
    - 시스템 전체의 하위 그룹 생성 신청을 상태별로 조회하고 관리(승인/거절)하는 기능.

## 6. 이메일 인증 (Email Verification) - `EmailVerificationController`

- **학교 이메일 인증:**
    - 사용자의 학교 이메일 주소로 인증 코드를 발송.
    - 수신한 코드를 검증하여 이메일 소유권을 확인하고, 사용자에게 '학교 인증' 상태를 부여.
