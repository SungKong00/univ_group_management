# React V0 개발 프롬프트: 5단계 - 사용자 개인화 기능 (프로필 및 알림)

## 1. 목표
사용자가 자신의 프로필 정보를 수정하고, 서비스 내의 중요한 활동에 대한 알림을 확인하는 기능을 구현합니다.

## 2. 선행 조건
-   사용자 인증 기능이 구현되어 있어야 합니다.

## 4. 상세 구현 단계

### 4.1. 사용자 프로필 관리
1.  **프로필 페이지 생성 (`src/pages/user/ProfilePage.tsx`)**
    -   `MainLayout` 내의 top-right에 위치한 사용자 아바타나 이름을 클릭했을 때 이동하는 페이지입니다.
    -   현재 로그인된 사용자의 프로필 정보(닉네임, 이메일, 프로필 이미지 등)를 API로 조회하여 보여줍니다.
    -   '수정하기' 버튼을 제공하여 프로필 수정 모드로 전환할 수 있게 합니다.

2.  **프로필 수정 기능**
    -   '수정하기' 모드에서는 닉네임, 프로필 이미지 등을 변경할 수 있는 입력 필드를 활성화합니다.
    -   '저장' 버튼 클릭 시, 프로필 수정 API (`/api/users/me`)를 호출하여 정보를 업데이트합니다.
    -   성공 시, "프로필이 업데이트되었습니다." 메시지를 보여주고, 화면의 프로필 정보를 갱신합니다. (예: `MainLayout`의 사용자 닉네임)

### 4.2. 알림 확인 기능
1.  **알림 아이콘 및 드롭다운 (`src/components/notification/NotificationDropdown.tsx`)**
    -   `MainLayout`의 헤더에 종 모양의 알림 아이콘(`NotificationsIcon`)을 배치합니다.
    -   읽지 않은 알림이 있을 경우, 아이콘에 빨간 점(MUI `Badge` 컴포넌트)을 표시합니다.
    -   아이콘 클릭 시, 최근 알림 목록을 보여주는 드롭다운 메뉴가 나타납니다.

2.  **알림 목록 조회**
    -   드롭다운이 열릴 때, 알림 목록 API (`/api/notifications`)를 호출하여 데이터를 가져옵니다.
    -   각 알림 항목은 내용과 수신 시간을 표시하며, 클릭 시 관련된 페이지(예: 새 댓글이 달린 게시글)로 이동해야 합니다.
    -   드롭다운 하단에 '모든 알림 보기' 링크를 두어 전체 알림 페이지로 이동시킵니다.

3.  **전체 알림 페이지 (`src/pages/notification/NotificationListPage.tsx`)**
    -   사용자가 받은 모든 알림을 시간순으로 보여주는 페이지입니다.
    -   알림을 읽음 처리하는 기능을 제공합니다.

## 5. 완료 조건
-   사용자는 자신의 프로필 페이지에서 정보를 확인하고 수정할 수 있다.
-   헤더의 알림 아이콘을 통해 최근 알림을 확인할 수 있다.
-   전체 알림 페이지에서 모든 알림 내역을 볼 수 있다.
