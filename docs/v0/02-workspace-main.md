# React V0 개발 프롬프트: 2단계 - 그룹 워크스페이스 조회 및 생성

## 1. 목표
로그인한 사용자가 자신이 속한 그룹(워크스페이스) 목록을 확인하고, 새로운 그룹을 생성하는 기능을 구현합니다. 이 단계는 서비스의 핵심인 그룹 활동의 시작점입니다.

## 2. 선행 조건
- 1단계에서 구현된 로그인 및 인증 기능이 정상적으로 동작해야 합니다.
- 로그인 시 발급된 JWT 토큰이 API 요청 헤더에 포함되어야 합니다.

## 4. 상세 구현 단계

### 4.1. 메인 대시보드 페이지 UI 구현
1.  **페이지 생성 (`src/pages/HomePage.tsx`)**
    -   로그인 후 처음으로 보여지는 메인 페이지입니다.
    -   페이지 상단에는 "내 워크스페이스"와 같은 제목과 "새 워크스페이스 생성" 버튼을 배치합니다.

2.  **워크스페이스 목록 컴포넌트 (`src/components/workspace/WorkspaceList.tsx`)**
    -   MUI의 `Grid`와 `Card` 컴포넌트를 사용하여 워크스페이스 목록을 표시합니다.
    -   각 카드는 워크스페이스의 이름, 설명, 멤버 수 등의 요약 정보를 보여줍니다.
    -   카드를 클릭하면 해당 워크스페이스의 상세 페이지(3단계에서 구현)로 이동해야 합니다. (`/workspace/{workspaceId}`)
    -   데이터를 불러오는 동안 `Skeleton` 또는 `CircularProgress` 컴포넌트를 사용하여 로딩 상태를 표시합니다.

### 4.2. API 연동 및 데이터 표시
1.  **워크스페이스 목록 조회**
    -   `HomePage`가 렌더링될 때, 백엔드 API (`/api/workspaces/my`)를 호출하여 사용자가 속한 워크스페이스 목록 데이터를 가져옵니다.
    -   `Zustand` 스토어(`src/store/workspaceStore.ts`)를 생성하여 워크스페이스 목록 상태를 관리합니다.
    -   API 호출 함수는 `src/api/workspaceApi.ts`에 작성합니다.

### 4.3. 새 워크스페이스 생성 기능 구현
1.  **생성 모달 (`src/components/workspace/CreateWorkspaceModal.tsx`)**
    -   "새 워크스페이스 생성" 버튼 클릭 시, MUI의 `Modal` 컴포넌트가 화면에 나타나야 합니다.
    -   모달 내부에는 워크스페이스 이름, 설명 등을 입력할 수 있는 폼이 포함됩니다.

2.  **생성 API 연동**
    -   폼 제출 시, 백엔드 API (`/api/workspaces`)를 호출하여 새 워크스페이스를 생성합니다.
    -   생성이 성공하면, 모달을 닫고 워크스페이스 목록을 다시 불러와 화면을 갱신합니다.
    -   사용자에게 "워크스페이스가 성공적으로 생성되었습니다."와 같은 `Snackbar` 알림을 표시합니다.

## 5. 완료 조건
-   로그인 후 메인 페이지에 접속하면, API를 통해 받아온 '내 워크스페이스' 목록이 카드 형태로 표시된다.
-   '새 워크스페이스 생성' 버튼을 누르면 생성 폼이 담긴 모달이 나타난다.
-   모달에서 정보를 입력하고 제출하면 워크스페이스가 실제로 생성되고, 목록이 자동으로 갱신된다.
-   워크스페이스 카드를 클릭하면, 아직 구현되지 않았더라도 콘솔에 해당 워크스페이스 ID가 출력되며 페이지 이동을 시도한다.
