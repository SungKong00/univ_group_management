# React V0 개발 프롬프트: 6단계 - 관리자 기능

## 1. 목표
서비스 운영자가 사용자와 그룹, 게시물 등을 관리할 수 있는 전용 관리자 페이지를 구현합니다.

## 2. 선행 조건
-   사용자 인증 기능이 구현되어 있어야 합니다.
-   API를 통해 현재 로그인한 사용자가 '관리자(ADMIN)' 역할인지 확인할 수 있어야 합니다.

## 3. 참고 명세
-   **기능 명세:** `docs/설계 문서/기능명세서/7 관리자 페이지.md`
-   **UI/UX 명세:** `docs/설계 문서/UI:UX 명세서/7. 관리자_페이지_ui_ux__1.md`

## 4. 상세 구현 단계

### 4.1. 관리자 페이지 접근 제어
1.  **관리자 전용 라우트 (`src/routes/AdminRoute.tsx`)**
    -   관리자 페이지(`/admin`)로 접근을 시도할 때, 사용자의 역할(role)을 확인하는 보호된 라우트 컴포넌트를 생성합니다.
    -   관리자 역할이 아닐 경우, 접근을 차단하고 홈페이지나 404 페이지로 리디렉션합니다.

### 4.2. 관리자 대시보드 및 레이아웃
1.  **관리자 레이아웃 (`src/layouts/AdminLayout.tsx`)**
    -   일반 사용자용 `MainLayout`과 별개로, 관리자 페이지를 위한 사이드바 메뉴가 포함된 레이아웃을 생성합니다.
    -   사이드바에는 '사용자 관리', '그룹 관리', '게시물 관리' 등의 메뉴 항목을 둡니다.

2.  **대시보드 페이지 (`src/pages/admin/DashboardPage.tsx`)**
    -   `/admin` 접속 시 처음 보여지는 페이지입니다.
    -   전체 사용자 수, 그룹 수, 오늘의 신규 가입자 등 주요 통계 지표를 요약하여 보여줍니다.

### 4.3. 관리 기능 구현
1.  **사용자 관리 페이지 (`src/pages/admin/UserManagementPage.tsx`)**
    -   MUI `DataGrid` 컴포넌트를 사용하여 전체 사용자 목록을 표시합니다. (페이지네이션 포함)
    -   사용자 검색 및 필터링 기능을 제공합니다.
    -   각 사용자에 대해 역할 변경, 계정 비활성화 등의 관리 작업을 수행할 수 있는 버튼/메뉴를 제공하고, 관련 API를 호출합니다.

2.  **그룹/게시물 관리 페이지**
    -   사용자 관리 페이지와 유사한 구조로, `DataGrid`를 사용하여 그룹 및 게시물 목록을 보여줍니다.
    -   부적절한 콘텐츠를 삭제하거나 숨김 처리하는 기능을 구현하고, 관련 API를 호출합니다.

## 5. 완료 조건
-   관리자 계정으로 로그인해야만 `/admin` 경로에 접근할 수 있다.
-   관리자 페이지는 전용 사이드바 메뉴가 있는 레이아웃을 사용한다.
-   관리자 대시보드에서 주요 통계 정보를 한눈에 볼 수 있다.
-   사용자 관리 페이지에서 사용자 목록을 조회하고 관리(역할 변경 등)할 수 있다.
