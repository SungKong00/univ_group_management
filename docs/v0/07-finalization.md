# React V0 개발 프롬프트: 7단계 - 전역 스타일링 및 배포 준비

## 1. 목표
개발된 애플리케이션의 전체적인 디자인 일관성을 확보하고, 반응형 디자인을 최종 점검하며, 실제 서비스 환경에 배포할 수 있도록 준비합니다.

## 2. 참고 명세
-   **아키텍처 설계:** `docs/설계 문서/아키텍쳐 설계 /배포 & ci cd.md`

## 3. 상세 구현 단계

### 3.1. 전역 스타일 및 테마 최종 점검
1.  **MUI 테마 커스터마이징 (`src/styles/theme.ts`)**
    -   `createTheme`을 사용하여 프로젝트의 기본 색상(primary, secondary), 폰트, 컴포넌트 기본 스타일(버튼, 카드 등)을 정의합니다.
    -   정의된 테마를 애플리케이션 최상단(`App.tsx` 또는 `main.tsx`)의 `ThemeProvider`에 적용합니다.

2.  **전역 CSS (`src/styles/global.css`)**
    -   필요한 경우, MUI 테마로 제어하기 어려운 전역 스타일(예: 스크롤바 디자인, 특정 라이브러리 오버라이드)을 정의합니다.

### 3.2. 공통 컴포넌트 및 유틸리티 점검
1.  **공통 컴포넌트 (`src/components/`)**
    -   프로젝트 전반에서 반복적으로 사용된 UI 요소(예: `PageTitle`, `ConfirmModal`)가 공통 컴포넌트로 잘 분리되어 있는지 확인하고, 일관된 `props` 인터페이스를 갖도록 리팩토링합니다.

2.  **에러 처리 및 로딩 상태**
    -   API 호출 시 발생할 수 있는 에러에 대한 전역 처리 로직을 `axios` 인터셉터에 추가합니다. (예: 401 에러 시 자동 로그아웃)
    -   데이터를 불러오는 동안 사용자에게 로딩 상태(스켈레톤 UI, 스피너 등)가 일관되게 표시되는지 전반적으로 검토합니다.

### 3.3. 반응형 디자인 최적화
1.  **레이아웃 및 컴포넌트**
    -   MUI의 `Grid`, `Stack`, `useMediaQuery` 훅 등을 사용하여 데스크톱, 태블릿, 모바일 화면 크기에 맞게 UI가 자연스럽게 변경되는지 확인합니다.
    -   특히 `DataGrid`, `Table`, 복잡한 폼 등이 모바일 환경에서 깨지지 않고 사용 가능하도록 스타일을 조정합니다.

### 3.4. 빌드 및 배포 준비
1.  **환경 변수 설정**
    -   개발(`.env.development`)과 프로덕션(`.env.production`) 환경에 따라 다른 API 기본 URL(`VITE_API_BASE_URL`)을 사용하도록 `.env` 파일을 설정합니다.
    -   `src/api/axiosInstance.ts`에서 `import.meta.env.VITE_API_BASE_URL`을 사용하도록 수정합니다.

2.  **빌드 스크립트 실행**
    -   `npm run build` 명령어를 실행하여 `dist` 디렉토리에 프로덕션용 정적 파일이 정상적으로 생성되는지 확인합니다.
    -   빌드 과정에서 발생하는 TypeScript 타입 에러나 ESLint 경고가 없는지 확인하고 수정합니다.

## 4. 완료 조건
-   애플리케이션 전체에 일관된 색상, 폰트, 컴포넌트 스타일이 적용된다.
-   모든 페이지가 모바일 화면 크기에서도 적절한 레이아웃으로 표시된다.
-   `npm run build` 명령어가 에러 없이 성공적으로 실행된다.
-   환경 변수를 통해 개발/프로덕션 API 서버 주소를 분리할 수 있다.
