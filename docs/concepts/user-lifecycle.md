# 사용자 여정 (User Lifecycle)

## 가입부터 활동까지의 여정

사용자가 플랫폼에 가입하여 활발하게 활동하기까지의 전체 과정을 안내합니다.

```
Google 로그인 → 프로필 작성 → 학과 자동 가입 → 그룹 활동 시작
    ↓
교수 역할 신청 (선택) → 승인 대기 → 교수 권한 획득
```

## 사용자 상태별 경험

### 1. 처음 방문하는 사용자
Google 계정으로 로그인한 직후의 상태입니다.

- **기본 정보**: Google에서 이메일, 이름 정보 자동 가져옴
- **접근 제한**: 프로필 완성 전까지는 제한된 접근만 가능
- **다음 단계**: 프로필 작성으로 자동 안내

### 2. 프로필 작성 중인 사용자
기본 정보를 입력하는 단계입니다.

- **필수 정보**: 닉네임, 학과, 학번 입력 필요
- **자동 안내**: 다른 페이지 접근 시 프로필 작성으로 리다이렉트
- **즉시 완성**: 모든 정보 입력 후 바로 플랫폼 이용 가능

### 3. 일반 학생 사용자
프로필 작성을 완료한 기본 사용자 상태입니다.

- **기본 활동**: 그룹 가입, 선택 학과 워크스페이스 활동, 권한이 필요없는 활동
- **소속 그룹**: 선택한 학과 그룹에 자동 가입
- **권한 범위**: 학생 수준의 기본 권한

### 4. 교수 승인 대기 중
교수 역할을 신청한 후 승인을 기다리는 상태입니다.

- **임시 권한**: 승인 전까지는 학생 권한으로 활동
- **신청 상태**: 프로필에 "교수 승인 대기 중" 표시
- **알림 받기**: 승인/거부 결과를 알림으로 확인

### 5. 승인된 교수
교수 역할이 승인되어 확장된 권한을 가진 상태입니다.

- **관리 권한**: 소속 학과 내 그룹들에 대한 관리 권한
- **교수 기능**: 학생들의 그룹 생성 승인, 감독 기능
- **높은 신뢰도**: 시스템 내에서 높은 권한과 책임

## 프로필 완성 과정

### 1. 간편한 Google 로그인
Google 계정으로 빠르고 안전하게 가입할 수 있습니다.

- **원클릭 로그인**: Google 계정 선택만으로 즉시 가입
- **정보 수동 입력**: 구글계정 가명일 수 있으니, 이름, 학과, 학번등은 직접 입력
- **보안**: Google의 보안 체계를 그대로 활용

### 2. 필수 정보 입력
플랫폼 이용을 위한 최소한의 정보를 입력합니다.

- **닉네임**: 플랫폼 내에서 사용할 별명
- **이름**: 실명 기입
- **학과 선택**: 소속 학과 선택 (목록에서 선택)
- **학번 입력**: 본인의 학번 입력
- **간단 완성**: 3-4개 항목만 입력하면 즉시 완료

### 3. 자동 학과 가입
프로필 완성과 동시에 자동으로 이루어지는 과정입니다.

- **즉시 가입**: 선택한 학과 그룹에 자동으로 가입
- **그룹 생성**: 해당 학과 그룹이 없으면 자동 생성
- **기본 멤버**: 학과 그룹의 일반 멤버로 시작
- **활동 시작**: 바로 학과 워크스페이스에 접근 가능

## 교수 승인 과정

### 1. 교수 역할 신청하기
학생 계정에서 교수 권한을 요청하는 과정입니다.

- **신청 이유**: 교수임을 증명할 간단한 사유 작성
- **신청 완료**: 프로필에서 "교수 승인 요청" 버튼 클릭
- **대기 상태**: 신청 후 관리자 승인까지 대기

### 2. 관리자 검토 및 승인
시스템 관리자가 교수 신청을 검토하는 단계입니다.

- **신청서 확인**: 신청자의 정보와 사유 검토
- **승인/거부**: 적절성 판단 후 승인 또는 거부 결정
- **처리 완료**: 결정 사항을 신청자에게 알림

### 3. 교수 권한 획득
승인 완료 시 자동으로 권한이 업그레이드됩니다.

- **즉시 적용**: 승인과 동시에 교수 권한 활성화
- **관리 권한**: 소속 학과의 그룹 관리 권한 획득
- **알림 확인**: "교수 계정이 승인되었습니다" 알림 수신

## 역할 변화에 따른 기능 변화

### 학생에서 교수로 승인될 때
교수 승인 시 사용할 수 있는 기능들이 크게 확대됩니다.

**새로 가능해지는 일들**:
- **그룹 관리**: 소속 학과 내 모든 그룹에 대한 관리 권한
- **그룹 승인**: 학생들의 새 그룹 생성 신청 승인/거부
- **교수 기능**: 학과 차원의 공지, 관리 업무 수행
- **높은 신뢰도**: 시스템 내에서 관리자급 권한과 책임
- **활동 통계 확인(추후 구현)**: 활동에 대한 통계 조회 가능

### 그룹 오너 권한 이양
그룹 오너가 그룹을 떠나거나 권한을 넘겨주는 상황입니다.

**이양 과정**:
1. **후계자 선택**: 현재 오너가 새로운 오너 지정
2. **권한 이전**: 모든 오너 권한이 새 오너에게 이전
3. **역할 변경**: 기존 오너는 일반 멤버가 되고, 새 오너는 모든 권한 획득
4. **알림 발송**: 그룹 멤버들에게 오너 변경 알림

## 계정 관리

### 일시적 계정 정지
휴학 등의 이유로 일시적으로 계정을 정지하는 기능입니다.

- **정지 사유**: 휴학, 교환학생 등 사유 기록
- **복귀 예정일**: 언제 돌아올 예정인지 설정
- **계정 보관**: 모든 데이터는 그대로 보관
- **복귀 시**: 정지 해제하면 모든 기능 즉시 복구

### 계정 완전 삭제
플랫폼을 더 이상 사용하지 않을 때의 처리입니다.

**삭제 전 처리 사항**:
1. **오너 권한 이양**: 소유한 그룹의 오너 권한을 다른 멤버에게 이양
2. **그룹 탈퇴**: 모든 가입 그룹에서 자동 탈퇴
3. **컨텐츠 익명화**: 작성한 게시글, 댓글은 "탈퇴한 사용자"로 표시
4. **완전 삭제**: 개인정보는 완전히 삭제

## 신입생을 위한 자동 설정

### 학과 자동 배정
신입생이 학과를 선택하지 않아도 시스템이 자동으로 처리합니다.

- **기본 학과**: 미선택 시 "AI/SW 학부"로 자동 배정
- **유연한 변경**: 나중에 올바른 학과로 변경 가능
- **즉시 참여**: 기본 배정되어도 바로 학과 활동 참여 가능

### 학과 구조 자동 생성
선택한 학과가 시스템에 없어도 자동으로 생성됩니다.

**자동 생성 과정 (예: AI/SW 학부 선택 시)**:
1. **대학교 확인**: "한신대학교" 최상위 그룹 확인/생성
2. **단과대학 생성**: "AI/SW 대학" 중간 그룹 생성
3. **학과 생성**: "AI/SW 학부" 최종 그룹 생성
4. **자동 가입**: 사용자를 해당 학과 그룹에 가입

## 사용자 상태별 화면 경험

### 프로필 미완성 시
프로필을 완성하지 않은 사용자의 경험입니다.

- **접근 제한**: 프로필 설정 화면만 접근 가능
- **자동 안내**: 다른 화면 접근 시 프로필 설정으로 리다이렉트
- **간단한 완성**: 몇 가지 정보만 입력하면 즉시 해결
- **즉시 이용**: 완성과 동시에 모든 기능 사용 가능

### 교수 승인 대기 시
교수 신청 후 승인을 기다리는 사용자의 경험입니다.

- **상태 표시**: 프로필 화면에 "교수 승인 대기 중" 배너 표시
- **임시 권한**: 승인 전까지는 학생 권한으로 활동
- **알림 대기**: 승인/거부 결과를 알림으로 확인
- **즉시 전환**: 승인 시 교수 기능 즉시 활성화
