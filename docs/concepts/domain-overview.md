# 도메인 개요 (Domain Overview)

## 시스템 목적

**대학 그룹 관리 시스템**은 대학 내 다양한 조직(학과, 동아리, 학회)이 효율적으로 소통하고 협업할 수 있도록 돕는 플랫폼입니다.

## 핵심 기능

### 사용자 온보딩
- **간편 가입**: Google 계정으로 간단하게 가입
- **자동 소속**: 학과 입력 시 해당 학과 그룹에 자동 가입
- **역할 관리**: 학생, 교수, 관리자 역할에 따른 차별화된 기능

### 그룹 관리
- **계층적 조직**: 대학 → 단과대 → 학과 → 동아리/학회 구조
- **유연한 그룹**: 공식 조직과 자율 모임 모두 지원
- **무제한 확장**: 그룹 내 하위 그룹 생성으로 세분화된 관리

### 협업 공간
- **전용 워크스페이스**: 각 그룹마다 독립적인 소통 공간
- **주제별 채널**: 목적에 맞는 다양한 대화 채널
- **권한 기반 접근**: 역할에 따른 세밀한 접근 제어
- **통합 캘린더**: 시간표, 개인/그룹 일정, 장소 예약 관리

### 멤버 모집
- **공개 모집**: 그룹 신규 멤버 모집 게시
- **체계적 관리**: 지원서 접수부터 승인까지 일괄 관리
- **자동 처리**: 승인 시 그룹 자동 가입

## 주요 구성 요소

### 사용자
- **다양한 역할**: 학생, 교수, 관리자가 각각 다른 기능 이용
- **간편한 프로필**: 학과, 학번 등 최소한의 정보로 시작
- **자동 연결**: 소속 학과 그룹에 즉시 참여

### 그룹
- **6가지 유형**: UNIVERSITY(대학교) / COLLEGE(단과대) / DEPARTMENT(학과) / LAB(연구실) / OFFICIAL(공식) / AUTONOMOUS(자율)
- **계층 관계**: 상위 그룹(대학)이 하위 그룹(학과/동아리) 관리
- **운영 방식**: 그룹 오너가 주도하고 멤버들이 참여

### 워크스페이스
- **전용 공간**: 각 그룹마다 독립적인 소통 공간
- **채널 시스템**: 주제별로 나뉜 대화방 (기본 2개 템플릿 제공)
- **캘린더 통합**: 개인/그룹 일정 및 장소 예약 관리
- **확장성**: 필요시 하위 그룹 생성으로 공간 확장

### 권한 관리
- **역할 기반**: 그룹 내 역할에 따른 기능 차별화
- **유연한 설정**: 그룹 특성에 맞는 권한 구조 설계

## 사용자 여정

### 신입생 가입 과정
1. **간편 로그인**: Google 계정으로 즉시 가입
2. **프로필 완성**: 학과, 학번 등 기본 정보 입력
3. **자동 소속**: 해당 학과 그룹에 자동 참여
4. **그룹 탐색**: 관심있는 동아리, 학회 찾아보기

### 그룹 만들기
1. **그룹 생성**: 새로운 동아리나 스터디 그룹 신청
2. **승인 대기**: 상위 그룹 관리자의 승인 기다림
3. **공간 확보**: 승인 시 전용 워크스페이스 자동 생성
4. **멤버 모집**: 신규 멤버 모집 게시글 작성

### 일상적 소통
1. **채널 활용**: 주제별 채널에서 대화 참여
2. **정보 공유**: 게시글 작성으로 정보 전달
3. **토론 참여**: 댓글로 의견 나누기
4. **공지 확인**: 중요한 공지사항 확인

### 그룹 가입하기
1. **모집 확인**: 관심 그룹의 모집 공고 확인
2. **지원서 작성**: 간단한 지원서 제출
3. **결과 대기**: 그룹 운영진의 심사 기다림
4. **활동 시작**: 승인 시 그룹 활동 즉시 시작

## 시스템 특징

### 사용자 친화적
- **직관적 구조**: 실제 대학 조직과 동일한 계층 구조
- **자동화**: 반복적 작업은 시스템이 자동 처리
- **유연성**: 다양한 그룹 형태와 운영 방식 지원

### 효율적 관리
- **역할 기반**: 각자 역할에 맞는 기능만 제공
- **권한 상속**: 상위 조직의 관리 권한이 하위까지 적용
- **체계적 승인**: 모든 변경사항은 적절한 승인 과정 거침
