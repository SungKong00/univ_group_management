# 워크스페이스와 채널 (Workspace & Channel)

그룹의 전용 소통 공간을 관리하는 시스템입니다.

## 개요

각 그룹은 **워크스페이스** 안에서 **채널**을 통해 체계적으로 소통합니다.

```
그룹 = 워크스페이스 (1:1 대응)
└── 📢 공지사항 (중요 알림)
    💬 일반대화 (자유로운 소통)
    💼 프로젝트 (주제별 채널)
```

## 워크스페이스

### 특징
- **자동 생성**: 그룹 생성 시 워크스페이스 자동 생성
- **1:1 대응**: 한 그룹 = 한 워크스페이스
- **기본 채널**: 공지사항, 자유게시판 자동 생성
- **생명 연동**: 그룹 삭제 시 워크스페이스도 함께 삭제

### 기본 채널

기본 2개 채널은 **자동으로 권한 템플릿 부여**:
- **공지사항**: 중요 알림용 (제한된 쓰기)
- **자유게시판**: 자유로운 소통 (모두 참여)

### 공간 확장

더 많은 소통 공간이 필요하면:
1. 특별한 프로젝트/이벤트용 **하위 그룹 생성**
2. 새로운 워크스페이스 자동 확보
3. 각각 **독립적으로 운영**

## 채널

### 유형
- **텍스트 채널**: 현재 지원하는 유일한 타입
  - 게시글 작성, 댓글, 파일 공유, 이모지 반응

- **공지 채널**: 중요 알림용 특별 표시 (향후 알림 연동)

### 채널 생성 과정

1. **권한 확인**: CHANNEL_MANAGE 권한 필요
2. **기본 정보 입력**: 채널 이름, 설명, 타입 선택
3. **초기 상태**: **권한 0개로 시작** (그룹장도 접근 불가)
4. **권한 설정**: Permission-Centric 매트릭스에서 권한별 허용 역할 지정
5. **CHANNEL_VIEW 부여**: 네비게이션에 표시되려면 CHANNEL_VIEW 권한 필수

### 권한 바인딩 정책

| 채널 | 초기 상태 | 권한 부여 |
|------|---------|---------|
| **기본 2채널** | 템플릿 자동 부여 | 수정 가능 |
| **사용자 정의** | 0개 (없음) | 수동으로 설정 필수 |

### 권한 설정 예시

```
CHANNEL_VIEW     → 그룹장, 멤버 (채널 네비게이션 노출)
POST_READ        → 그룹장, 멤버 (게시글 조회)
POST_WRITE       → 그룹장 (게시글 작성)
COMMENT_WRITE    → 그룹장 (댓글 달기)
FILE_UPLOAD      → 그룹장 (선택)
```

## 콘텐츠 구조

### 게시글
- **제목**: 선택사항
- **본문**: 필수
- **파일**: 첨부 가능 (권한 허용 시)
- **상단 고정**: 중요한 글은 채널 상단에 고정
- **조회수 추적**: 몇 명이 읽었는지 확인

### 댓글
- **댓글**: 게시글에 대한 의견/질문
- **대댓글**: 다른 댓글에 답글 (최대 2단계)
- **시간 순서**: 작성 시간순 정렬
- **활발한 토론**: 여러 사람 참여 가능

## 실제 사용 사례

### 동아리 운영
```
프로그래밍 동아리
├─ 📢 공지사항 (회장만 작성)
├─ 💬 자유게시판 (모두 참여)
└─ 💼 스터디 (스터디 전담 채널)
```

### 프로젝트 팀
```
해커톤 팀
├─ 💭 아이디어 논의 (기획)
├─ 👨‍💻 개발 진행상황 (구현)
├─ 📋 회의록 (기록)
└─ 🐛 버그 리포트 (이슈 추적)
```

### 신입 모집
```
신입 모집
├─ 📝 지원서 접수 (Q&A)
├─ ❓ 질문 답변 (FAQ)
└─ 📅 면접 일정 (공지용)
```
