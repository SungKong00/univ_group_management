# 워크스페이스와 채널 (Workspace & Channel)

## 그룹의 전용 소통 공간

모든 그룹은 자신만의 워크스페이스를 가지며, 이 공간에서 체계적으로 소통할 수 있습니다.

```
그룹 = 워크스페이스 (1:1 대응)
└── 그룹 전용 워크스페이스
    ├── 📢 공지사항 (중요한 알림)
    ├── 💬 일반대화 (자유로운 소통)
    └── 💼 프로젝트 논의 (주제별 채널)

공간이 더 필요하면:
그룹 A → 하위 그룹 생성 → 새로운 워크스페이스 확보
```

## 워크스페이스의 특징

### 그룹과 1:1 연결
각 그룹은 반드시 하나의 워크스페이스를 가집니다.

- **자동 생성**: 그룹 만들어지면 워크스페이스도 자동으로 생성
- **같은 이름**: 그룹 이름과 워크스페이스 이름이 동일
- **기본 구성**: 공지사항, 일반대화 채널이 자동으로 준비됨
- **생명 연동**: 그룹이 사라지면 워크스페이스도 함께 사라짐

### 공간 확장 방법
더 많은 소통 공간이 필요할 때의 해결책입니다.

- **하위 그룹 생성**: 특별한 프로젝트나 이벤트를 위한 새 그룹 만들기
- **독립적 운영**: 각 워크스페이스는 독립적으로 관리
- **유연한 활용**: 필요에 따라 그룹을 늘리거나 줄일 수 있음

## 채널의 종류

### 텍스트 채널 - 모든 소통의 기본
현재 사용할 수 있는 유일한 채널 타입입니다.

**할 수 있는 일들**:
- **게시글 작성**: 긴 글이나 공지사항 작성
- **댓글 달기**: 다른 사람 글에 의견 남기기
- **파일 공유**: 문서, 이미지 등 파일 첨부
- **이모지 반응**: 간단한 감정 표현

**활용 방법**:
- **일반 대화**: 자유로운 소통과 잡담
- **질문 답변**: 궁금한 점 질문하고 답변받기
- **토론**: 주제에 대한 깊이 있는 논의
- **공지사항**: 중요한 알림과 공지

### 채널 구분 방법
채널의 용도를 명확하게 구분할 수 있습니다.

- **일반 채널**: 기본적인 텍스트 채널
- **공지 채널**: 중요한 알림을 위한 특별 표시 (향후 알림 기능 연동 예정)

### 향후 추가 예정
앞으로 구현될 채널 타입들입니다.

- **음성 채널**: 실시간 음성 대화, 화면 공유 (우선순위 낮음)
- **파일 공유**: 현재는 텍스트 채널 내 파일 첨부로 충분

## 채널 접근 관리

### 권한에 따른 자동 제어
복잡한 설정 없이 권한만으로 채널 접근을 관리합니다.

**보이는 채널만 접근 가능**:
- 권한이 있는 채널만 네비게이션에 표시
- 권한이 없는 채널은 존재조차 알 수 없음
- 별도의 공개/비공개 설정 불필요

### 채널별 권한 종류
각 채널에서 할 수 있는 일들을 세밀하게 조정할 수 있습니다.

- **보기 권한**: 채널 존재 확인 및 입장
- **읽기 권한**: 채널 내 게시글과 댓글 읽기
- **쓰기 권한**: 새 게시글이나 댓글 작성
- **관리 권한**: 채널 설정 변경
- **삭제 권한**: 채널 삭제

**간단한 원칙**:
- 보기 권한이 없으면 채널이 보이지 않음
- 각 기능별로 세밀한 권한 조정 가능
- 권한 설정만으로 모든 접근 제어 완료

## 채널 내 콘텐츠

### 게시글로 정보 공유
채널의 주요 콘텐츠는 게시글입니다.

**게시글의 특징**:
- **제목과 본문**: 제목은 선택사항, 본문은 필수
- **다양한 용도**: 일반 글, 공지사항, 질문, 투표 등
- **상단 고정**: 중요한 글은 채널 상단에 고정 가능
- **조회수 확인**: 얼마나 많은 사람이 읽었는지 확인

### 댓글로 소통하기
게시글에 대한 반응과 토론은 댓글로 이루어집니다.

**댓글 시스템**:
- **댓글 달기**: 게시글에 대한 의견이나 질문 작성
- **대댓글**: 다른 댓글에 답글 달기 (최대 2단계까지)
- **활발한 토론**: 여러 사람이 참여하는 토론 진행
- **시간 순서**: 작성 시간 순으로 정렬되어 대화 흐름 파악 용이

## 사용자 화면 구성

### 채널 목록 화면
워크스페이스 왼쪽에 표시되는 채널 목록입니다.

```
📢 공지사항 (중요한 알림)
💬 일반대화 (자유로운 소통)
💬 자유게시판 (자유 주제)
💬 운영진전용 (제한된 접근)
```

### 게시글 목록 화면
선택한 채널의 게시글들을 보여주는 메인 화면입니다.

```
[📌 고정] 신입 모집 안내 (상단 고정된 중요 글)
[👤 김철수] 스터디 참여자 모집
[👤 이영희] 프로젝트 아이디어 공유
[👤 박민수] 질문이 있습니다
```

### 실시간 대화 형태
빠른 소통을 위한 채팅 스타일 화면입니다.

```
김철수 | 오후 2:30
안녕하세요! 신입 가입 문의드립니다.

이영희 | 오후 2:31
@김철수 환영합니다! 가입 절차는...

[새 메시지 입력하기]
```

## 새 채널 만들기

### 채널 생성 과정
새로운 채널은 간단한 과정으로 만들 수 있습니다.

1. **권한 확인**: 채널 관리 권한이 있는 사용자만 생성 가능
2. **기본 정보 입력**: 채널 이름, 설명, 용도 등 입력
3. **자동 설정**: 기본 권한과 설정이 자동으로 적용
4. **즉시 사용**: 생성과 동시에 멤버들이 접근 가능

### 채널 설정 항목
새 채널을 만들 때 설정할 수 있는 항목들입니다.

- **채널 이름**: "개발논의", "자유게시판" 등 목적을 나타내는 이름
- **채널 설명**: 어떤 용도로 사용하는지 간단한 설명
- **채널 타입**: 일반 채널 또는 공지 채널 선택
- **접근 권한**: 어떤 역할의 멤버가 접근할 수 있는지 설정

### 자동으로 설정되는 것들
채널 생성 시 자동으로 처리되는 사항들입니다.

- **기본 권한**: 그룹 오너와 멤버에 대한 기본 권한 자동 설정
- **권한 바인딩**: 역할별로 적절한 채널 권한 자동 연결
- **가시성 제어**: 권한이 있는 멤버에게만 채널이 보이도록 설정

## 실제 활용 사례

### 동아리 기본 운영
**상황**: 프로그래밍 동아리의 일상적인 운영

1. **동아리 생성**: "프로그래밍 동아리" 그룹 생성
2. **워크스페이스 확보**: 동아리 전용 워크스페이스 자동 생성
3. **기본 채널 활용**:
   - 📢 **공지사항**: 중요한 알림 (회장만 작성 가능)
   - 💬 **자유게시판**: 자유로운 소통 (모든 멤버 참여)
4. **일상 운영**: 공지, 토론, 정보 공유 등 활발한 활동

### 프로젝트 전담 공간
**상황**: 해커톤 참가를 위한 별도 팀 구성

1. **전담 그룹 생성**: "해커톤 프로젝트팀" 하위 그룹 생성
2. **전용 워크스페이스**: 프로젝트만을 위한 독립 공간 확보
3. **목적별 채널 구성**:
   - 💭 **아이디어 논의**: 브레인스토밍과 기획
   - 👨‍💻 **개발 진행상황**: 작업 진도 공유
   - 📋 **회의록**: 회의 내용 기록
   - 🐛 **버그 리포트**: 문제점 공유 및 해결
4. **프로젝트 종료**: 완료 후 그룹 삭제 또는 다른 용도로 전환

### 신입 모집 시즌
**상황**: 동아리 신입 모집을 위한 전담 운영

1. **모집 전담 그룹**: "신입 모집" 하위 그룹 생성
2. **모집 워크스페이스**: 모집 업무 전용 공간
3. **모집 특화 채널**:
   - 📝 **지원서 접수**: 지원자들이 지원서 제출
   - ❓ **질문답변**: 지원자들의 궁금증 해결
   - 📅 **면접 일정**: 면접 일정 공지 (운영진만 작성)
4. **모집 마무리**: 모집 종료 후 그룹 정리

### 대규모 이벤트 조직
**상황**: 학과 축제 같은 큰 행사 준비

1. **메인 그룹**: "학과 축제" 이벤트 그룹 생성
2. **팀별 하위 그룹**: 업무 분담을 위한 세부 팀 구성
   - **기획위원회** → 기획 전용 워크스페이스
   - **대외협력팀** → 대외업무 전용 워크스페이스
   - **현장운영팀** → 현장관리 전용 워크스페이스
3. **독립적 운영**: 각 팀이 자신만의 공간에서 효율적으로 작업
4. **통합 관리**: 필요시 메인 그룹에서 전체 조율
