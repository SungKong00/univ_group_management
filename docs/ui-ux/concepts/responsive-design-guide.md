# 반응형 UI 가이드: 레이아웃 및 상태 유지

본 문서는 웹(넓은 화면)과 모바일(좁은 화면) 사이를 전환할 때 일관된 사용자 경험을 제공하기 위한 레이아웃 원칙과 상태 유지 시나리오를 정의합니다. 프로젝트의 모든 페이지는 이 가이드를 따라야 합니다.

## 1. 기본 레이아웃 원칙

### 1.1. 웹 (넓은 화면)

-   **2단 레이아웃**: 일반적으로 **좌측 네비게이션 바**와 **중앙 콘텐츠 영역**으로 구성된 2단 레이아웃을 사용합니다.
-   **분할 뷰 (Split View)**: 필요에 따라 콘텐츠 영역을 목록과 상세 보기로 나누는 분할 뷰를 사용할 수 있습니다. (예: 게시글 목록과 댓글 사이드바)

### 1.2. 모바일 (좁은 화면)

-   **하단 네비게이션 바**: 웹의 좌측 네비게이션은 모바일에서 **하단 탭 네비게이션**으로 대체됩니다.
-   **전체 화면 뷰 (Full-Screen View)**: 웹에서 한 화면에 보이던 여러 영역(목록, 상세, 댓글 등)은 모바일에서 각각의 전체 화면으로 분리됩니다. 사용자는 터치를 통해 각 화면을 오가게 됩니다.

## 2. 핵심 원칙: 상태 유지 (State Preservation)

화면 레이아웃이 변경될 때 사용자의 현재 **컨텍스트(State)**는 절대 소실되어서는 안 됩니다. '상태'란 사용자가 보고 있던 화면, 스크롤 위치, 입력 중인 데이터 등 모든 상호작용의 맥락을 의미합니다.

UI가 화면 크기에 맞춰 재구성될 때 상태가 사라지면, 사용자는 하던 작업을 처음부터 다시 시작해야 하는 불편함을 겪습니다. 이를 방지하기 위해, 개발자는 아래 시나리오들을 참고하여 모든 상태가 매끄럽게 보존되도록 구현해야 합니다.

## 3. 상세 전환 시나리오

### 3.1. 웹 → 모바일 전환

-   **시나리오 1: 리스트(게시글 등)를 보던 중**
    -   **기대 동작**: 모바일 뷰 전환 시, 상위 목록이 아닌 **현재 보던 리스트**가 전체 화면으로 표시되어야 합니다. 스크롤 위치도 최대한 유지되어야 합니다.

-   **시나리오 2: 분할 뷰(예: 게시글과 댓글)를 보던 중**
    -   **기대 동작**: 웹에서 댓글 사이드바를 열어 둔 상태였다면, 모바일 전환 시 **댓글 뷰가 전체 화면으로** 바로 표시되어야 합니다.

-   **시나리오 3: 입력 필드에 텍스트를 작성하던 중**
    -   **기대 동작**: 모바일 뷰로 전환되어도, **작성 중이던 텍스트가 입력창에 그대로 유지**되어야 합니다.

-   **시나리오 4: 탭(Tab)으로 구분된 화면을 보던 중**
    -   **기대 동작**: 모바일 뷰 전환 시, **이전에 활성화했던 탭이 그대로 선택된 상태**로 표시되어야 합니다.

### 3.2. 모바일 → 웹 전환

-   **시나리오 1: 모바일에서 상세 정보/댓글 뷰를 보던 중**
    -   **기대 동작**: 웹 뷰로 전환 시, 목록과 상세 정보/댓글이 함께 표시되는 **분할 화면 레이아웃**이 나타나야 합니다. 즉, **댓글 사이드바가 열린 상태의 웹 뷰**가 표시되어야 합니다.

-   **시나리오 2: 모바일에서 특정 탭을 보던 중**
    -   **기대 동작**: 웹 뷰로 전환 시, **해당 탭이 그대로 선택된 상태**로 표시되어야 합니다.

## 4. 뒤로가기 동작의 일관성

-   뷰 전환 후에도 뒤로가기 동작의 일관성은 유지되어야 합니다.
-   모바일에서는 화면 단위로 뒤로가기가 동작해야 합니다. 예를 들어, `댓글 뷰` → (뒤로가기) → `게시글 뷰` → (뒤로가기) → `채널 목록 뷰` 순서로 동작해야 합니다.
