# UI/UX 명세서: 인증 페이지 (로그인 & 가입)

본 문서는 사용자의 로그인 및 신규 가입 과정에 대한 UI/UX 정책을 정의합니다. 토스의 디자인 철학에 따라, 사용자가 하나의 핵심 목표에만 집중할 수 있도록 간결하고 명확하게 설계합니다.

## 1. 로그인 페이지

로그인 페이지는 사용자가 서비스를 시작하는 첫 관문입니다. 복잡한 입력 없이, 익숙한 소셜 로그인을 통해 즉시 서비스를 시작할 수 있도록 유도합니다.

### 1.1. 디자인 컨셉

-   **화면 구성**: 웹과 모바일 모두 화면 중앙에 모든 요소를 배치하여 사용자의 시선을 자연스럽게 집중시킵니다.
-   **핵심 행동 유도**: "Google로 계속하기"라는 명확한 Call-to-Action(CTA) 버튼 하나만을 제시하여 고민의 여지를 없앱니다.
-   **개발 버전**: 개발 단계에서는 간편한 로그인을 위해서 관리자 계정으로 로그인 버튼을 만들어 넣는다.

### 1.2. 구성 요소 (위에서 아래로)

1.  **로고**: 앱의 심볼을 간결하게 표시합니다.
2.  **앱 이름**: 로고 아래에 명확하고 굵은 폰트로 표시합니다.
3.  **소개 문구**: 앱의 핵심 가치를 한두 줄의 간결한 문장으로 전달합니다. (예: "우리 학과 학생들을 위한 똑똑한 협업 공간")
4.  **CTA 버튼**:
-   **텍스트**: "Google로 계속하기"
-   **디자인**: 화면 하단에 구글 로고와 함께 배치되며, 사용자가 가장 쉽게 누를 수 있는 크기와 형태로 제공됩니다.

### 1.3. 사용자 플로우

1.  사용자가 "Google로 계속하기" 버튼을 클릭합니다.
2.  시스템은 Google 계정 정보를 기반으로 가입 여부를 확인합니다.
-   **기 가입 사용자**: 즉시 **홈 화면**으로 이동합니다.
-   **신규 사용자**: 아래의 **가입 페이지**로 자동 안내됩니다.

## 2. 가입 페이지 (온보딩)

신규 사용자를 위한 가입 페이지는 `user-lifecycle.md`와 `register_screen.dart` 구현을 참고하여, 최소한의 정보만으로 빠르게 가입을 완료할 수 있도록 설계합니다.

### 2.1. 디자인 컨셉

-   **단계적 정보 입력**: 사용자가 부담을 느끼지 않도록 정보를 논리적 순서에 따라 입력받습니다.
-   **명확한 안내**: 각 입력 필드마다 어떤 정보를 왜 입력해야 하는지 명확한 `Label`과 `Hint`를 제공합니다.
-   **즉각적인 피드백**: 입력값의 유효성을 실시간으로 검증하여 사용자에게 즉시 피드백(성공, 오류, 제안)을 줍니다. (예: 닉네임 중복 검사)

### 2.2. 입력 정보 및 플로우

가입 페이지는 "기본 정보를 입력해주세요"라는 큰 제목 아래 다음 정보를 순서대로 입력받습니다.

1.  **이름**:
-   `Label`: 이름
-   `Hint`: 실명
-   Google 계정의 이름을 기본으로 채워주지만, 수정 가능해야 합니다.

2.  **닉네임**:
-   `Label`: 닉네임
-   `Hint`: 다른 사용자에게 표시될 이름
-   **실시간 검증**: 2자 이상, 유효한 문자(한/영/숫자) 규칙을 검사하고, 중복 여부를 즉시 확인하여 "사용 가능한 닉네임이에요" 또는 "이미 사용 중인 닉네임입니다" 와 같은 피드백을 제공합니다. 중복 시, 추천 닉네임을 `Chip` 형태로 제안합니다.

3.  **소속 (계열/학과)**:
-   `Label`: 계열, 학과 (선택)
-   계열을 먼저 선택하면, 해당 계열에 속한 학과 목록이 활성화되는 2단계 드롭다운 메뉴를 제공합니다.
-   계열(단과대)는 선택 필수, 학과는 선택임. 계열(단과대)에만 소속된 학생이 있기 때문임.

4.  **학적 정보 (학번, 학년)**:
-   `Label`: 학번, 학년
-   `Hint`: 예: 20251234, 1학년
-   학번과 학년(드롭다운)을 필수로 입력받습니다. 와이드 화면에서는 두 필드를 한 줄에 나란히 표시합니다.

5.  **역할 선택**:
-   `Label`: 어떤 역할로 시작하시겠어요?
-   '학생'과 '교수'를 선택할 수 있는 `SegmentedButton`을 제공합니다.
-   '교수' 선택 시, "교수 권한은 관리자 승인 후 활성화돼요"라는 안내 문구를 표시합니다.

6.  **학교 이메일 인증**:
-   `Label`: 학교 이메일
-   `.ac.kr`로 끝나는 이메일 주소를 입력받습니다.
-   '인증 코드 받기' 버튼을 누르면 해당 이메일로 6자리 OTP 코드가 발송됩니다.
-   **OTP 입력**: 코드 입력 시 타이머(5분)가 표시되며, '재전송' 버튼을 제공합니다. 코드가 확인되면 "인증이 완료되었어요" 피드백을 줍니다.

7.  **제출 버튼**:
-   **텍스트**: "완료"
-   모든 필수 정보(이름, 닉네임, 계열, 이메일 인증)가 올바르게 입력되기 전까지는 **비활성화** 상태를 유지합니다.
-   버튼 클릭 시, 로딩 상태를 표시하며 프로필 생성을 요청하고, 성공 시 **홈 화면**으로 이동합니다.
