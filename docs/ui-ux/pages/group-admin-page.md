# UI/UX 명세서: 그룹 관리 페이지

본 문서는 그룹 관리자가 그룹의 설정을 변경하고, 멤버를 관리하며, 역할을 편집하는 등 그룹 운영 전반에 필요한 기능을 수행하는 '그룹 관리' 페이지의 UI/UX를 정의합니다.

## 1. 접근 조건

-   **권한**: 현재 그룹에 대해 `GROUP_MANAGE` 또는 그 이상의 관리 권한을 가진 사용자만 접근할 수 있습니다.
-   **진입점**: 워크스페이스의 채널 네비게이션 바 하단에 위치한 '관리자 페이지' 버튼을 통해 진입합니다. 권한이 없는 사용자에겐 이 버튼이 표시되지 않습니다.

## 2. 페이지 레이아웃

그룹 관리 페이지는 좌측에 서브-네비게이션 메뉴를, 우측에 선택된 메뉴에 해당하는 콘텐츠 영역을 가집니다.

-   **좌측 서브-네비게이션**:
    -   일반 설정
    -   멤버 관리
    -   역할 및 권한
    -   가입 신청 관리
    -   통계 (구현 예정)
-   **우측 콘텐츠 영역**: 선택된 서브 메뉴의 내용이 표시됩니다.

## 3. 주요 기능별 명세

### 3.1. 일반 설정

-   **기능**: 그룹의 이름, 설명, 공개 범위, 프로필 이미지 등 기본 정보를 수정할 수 있습니다.
-   **UI**:
    -   각 설정 항목은 명확한 레이블과 함께 입력 필드 또는 드롭다운 메뉴로 제공됩니다.
    -   [저장] 버튼을 눌러야 변경사항이 적용되며, 변경 전까지는 비활성화 상태입니다.

### 3.2. 멤버 관리

-   **기능**: 그룹에 속한 모든 멤버의 목록을 확인하고, 역할을 변경하거나 멤버를 강제로 탈퇴시킬 수 있습니다.
-   **UI**:
    -   멤버 목록은 테이블 형태로 표시되며, 각 행은 사용자 프로필, 이름, 현재 역할, 가입일 등의 정보를 포함합니다.
    -   **역할 변경**: 각 멤버 행의 '역할' 항목은 드롭다운 메뉴로 되어 있어, 즉시 다른 역할로 변경할 수 있습니다.
    -   **강제 탈퇴**: 각 멤버 행 끝에 있는 [...] 메뉴를 통해 '강제 탈퇴' 옵션을 선택할 수 있습니다. 실행 시, 확인 모달이 표시됩니다.
    -   상단에는 이름이나 역할로 멤버를 검색할 수 있는 검색창이 제공됩니다.

### 3.3. 역할 및 권한

-   **기능**: 그룹 내 역할을 생성, 수정, 삭제하고 각 역할에 부여된 권한을 관리합니다.
-   **UI**:
    -   역할 목록이 좌측에 표시되며, 선택된 역할의 상세 권한 설정이 우측에 나타납니다.
    -   **권한 설정**: 권한은 트리 또는 체크박스 목록 형태로 그룹화되어 제공됩니다. (예: 그룹 관리, 멤버 관리, 채널 관리 등)
    -   **시스템 역할**: OWNER, ADVISOR, MEMBER와 같은 시스템 역할은 삭제하거나 일부 중요 권한을 변경할 수 없도록 비활성화 처리됩니다.

### 3.4. 가입 신청 관리

-   **기능**: 그룹 가입을 신청한 사용자 목록을 확인하고, 신청을 승인하거나 거절합니다.
-   **UI**:
    -   가입 신청 목록이 테이블 형태로 표시됩니다.
    -   각 신청 항목에는 신청자의 정보와 함께 [승인], [거절] 버튼이 제공됩니다.
    -   일괄 처리(전체 선택 후 승인/거절) 기능이 제공될 수 있습니다.
