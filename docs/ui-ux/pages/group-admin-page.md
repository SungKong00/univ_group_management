# 그룹 관리 페이지 (Group Admin Page)

> **소유자**: `frontend-specialist`
> **상태**: `기능 구현 중`

이 문서는 그룹 관리 페이지의 UI/UX 및 기능 명세를 정의합니다.

## 1. 페이지 개요

-   **경로**: 워크스페이스 내 '그룹 관리' 뷰
-   **진입점**: 워크스페이스 좌측 채널 목록 하단의 "그룹 관리" 버튼
-   **접근 권한**: `GROUP_MANAGE`, `MEMBER_MANAGE`, `CHANNEL_MANAGE`, `RECRUITMENT_MANAGE` 권한 중 하나 이상을 보유한 사용자

## 2. UI 레이아웃

페이지는 반응형 레이아웃을 따릅니다. 데스크톱 화면에서는 관리 섹션들이 2열 그리드 형태로 표시되어 공간 활용도를 높이고, 모바일 화면에서는 기존의 세로 목록 형태로 표시됩니다.

- **루트 위젯**: `Scaffold` > `LayoutBuilder` > `Wrap` (데스크톱) or `ListView` (모바일)
- **섹션 위젯**: `_AdminSection` (제목, 설명, 아이콘 포함)
- **액션 위젯**: `ActionCard` (각 기능으로 연결되는 버튼 카드)

## 3. 기능 섹션 및 동작

### 3.1. 그룹 설정

- **아이콘**: `Icons.settings_outlined`
- **설명**: 그룹 정보 및 주요 설정을 관리합니다.
- **액션**:
    -   **그룹 정보 수정**:
        -   **동작**: `showEditGroupDialog`를 호출하여 다이얼로그를 엽니다.
        -   **기능**: 그룹 이름, 설명, 모집 상태, 태그 등을 수정할 수 있습니다.
        -   **상태**: **구현 완료**
    -   **공개 범위 설정**:
        -   **동작**: "준비 중" 다이얼로그 표시.
        -   **비고**: 백엔드에서 `GroupVisibility` 개념이 제거됨에 따라 이 기능은 **폐기될 예정**입니다.
        -   **상태**: `구현 예정 없음`
    -   **그룹 삭제**:
        -   **동작**: "준비 중" 다이얼로그 표시.
        -   **상태**: `향후 구현 예정`

### 3.2. 멤버 관리

- **아이콘**: `Icons.people_outline`
- **설명**: 그룹 멤버 초대 및 역할을 관리합니다.
- **액션**:
    -   **멤버 초대**:
        -   **동작**: "준비 중" 다이얼로그 표시.
        -   **상태**: `향후 구현 예정`
    -   **멤버 목록**:
        -   **동작**: `WorkspaceView`를 `memberManagement`로 변경하여 멤버 관리 페이지로 이동합니다.
        -   **상태**: **구현 완료**
    -   **역할 관리**:
        -   **동작**: "준비 중" 다이얼로그 표시.
        -   **상태**: `향후 구현 예정`

### 3.3. 채널 관리

- **아이콘**: `Icons.tag_outlined`
- **설명**: 워크스페이스 채널 생성 및 설정을 관리합니다.
- **액션**:
    -   **채널 목록 및 권한 관리**:
        -   **동작**: `WorkspaceView`를 `channelManagement`로 변경하여 채널 관리 페이지로 이동합니다.
        -   **상태**: **구현 완료**
    -   **채널 생성**:
        -   **동작**: "준비 중" 다이얼로그 표시.
        -   **상태**: `향후 구현 예정`

### 3.4. 모집 관리

- **아이콘**: `Icons.campaign_outlined`
- **설명**: 신규 멤버 모집 및 지원자를 관리합니다.
- **액션**:
    -   **모집 공고 작성**:
        -   **동작**: "준비 중" 다이얼로그 표시.
        -   **상태**: `향후 구현 예정`
    -   **지원자 관리**:
        -   **동작**: `WorkspaceView`를 `applicationManagement`로 변경하여 지원자 관리 페이지로 이동합니다.
        -   **상태**: **구현 완료**

## 4. 주요 상호작용

-   **그룹 정보 수정**: 사용자가 `ActionCard`를 탭하면, 현재 그룹 정보를 바탕으로 `EditGroupDialog`가 나타납니다. 수정을 완료하면 `myGroupsProvider`와 `workspaceStateProvider`가 갱신되어 변경사항이 즉시 반영됩니다.
-   **멤버 목록 보기**: 사용자가 `ActionCard`를 탭하면, `workspaceStateProvider`의 상태가 변경되어 `WorkspacePage`의 메인 컨텐츠가 `MemberManagementPage`로 교체됩니다.

## 5. 리팩토링 및 코드 구조

-   페이지 내에서 사용되던 `_ActionCard` 위젯은 공용 위젯인 `widgets/cards/action_card.dart`로 분리되어 재사용성이 향상되었습니다. 이 카드는 `isDestructive` 속성을 통해 삭제와 같은 위험한 작업을 시각적으로 구분할 수 있습니다.
