# UI/UX 명세서: 모집 페이지

본 문서는 그룹의 신규 멤버 모집과 관련된 전체 페이지의 구조와 사용자 경험(UX)을 정의합니다. 사용자는 이 페이지들을 통해 모집 공고를 확인하고, 지원서를 제출하며, 관리자는 지원자를 관리합니다.

## 페이지 네비게이션 구조

### 접근 경로
- **일반 사용자**: 홈 > 그룹 상세 > 모집 공고 (공개)
- **관리자**: 워크스페이스 > 관리자 메뉴 > 모집 관리

### 관리자 페이지 구조 (탭 기반)
관리자 메뉴의 "모집 관리" 버튼 클릭 시 다음과 같은 탭 구조로 표시:

```
[모집 공고 관리] [지원자 관리]
─────────────────────────────────
(현재 탭 콘텐츠)
```

**기술 구현**:
- `TabController` 사용 (멤버 관리 페이지와 동일한 패턴)
- 반응형 분기: `ResponsiveBreakpoints.of(context).largerThan(MOBILE)` (768px 기준)
- 데스크톱: 테이블 레이아웃
- 모바일: 카드 레이아웃

## 1. 모집 공고 리스트 페이지

사용자가 현재 진행 중인 모든 그룹의 모집 공고를 탐색하는 페이지입니다.

### 1.1. 디자인 컨셉

-   **탐색의 용이성**: 사용자가 원하는 그룹의 모집 공고를 쉽게 찾을 수 있도록 카드 기반 레이아웃과 필터/검색 기능을 제공합니다.
-   **정보의 명확성**: 각 모집 공고의 핵심 정보(그룹명, 모집 제목, 마감일, 지원자 수)를 한눈에 파악할 수 있도록 `Card` 컴포넌트를 활용합니다.

### 1.2. 구성 요소

1.  **페이지 제목**: "모집"
2.  **검색 바**:
    -   `Text Field` 컴포넌트를 사용하며, `Hint` 텍스트로 "그룹명 또는 모집 제목으로 검색"을 표시합니다.
    -   실시간으로 입력된 키워드에 맞는 모집 공고만 필터링하여 보여줍니다.
3.  **모집 공고 카드 리스트**:
    -   각 모집 공고는 하나의 `Card` 컴포넌트로 표현됩니다.
    -   **카드 내용 (위에서 아래로)**:
        -   **그룹 정보**: 그룹 로고 이미지와 그룹명
        -   **모집 제목**: 굵은 폰트로 강조
        -   **마감 정보**: `Chip` 컴포넌트를 사용하여 "N일 남음" 또는 "오늘 마감" 등 마감까지 남은 기간을 표시합니다.
        -   **지원 현황**: (공개 설정 시) 👥 아이콘과 함께 "현재 N명 지원" 텍스트를 표시합니다.
    -   카드를 클릭하면 아래 `2. 모집 공고 상세 페이지`로 이동합니다.

## 2. 모집 공고 상세 페이지

사용자가 특정 모집 공고의 상세 내용을 확인하고 지원서를 작성하는 페이지입니다.

### 2.1. 기본 구조

-   **헤더**: `Top App Bar`를 사용하여 뒤로가기 버튼과 모집 공고 제목을 표시합니다.
-   **콘텐츠 영역**: 스크롤 가능한 영역에 상세 내용과 지원서 폼이 위치합니다.
-   **하단 고정 버튼**: 화면 하단에 `Filled Button`으로 "지원하기" CTA 버튼을 고정하여 사용자가 언제든 지원 절차를 시작할 수 있도록 합니다.

### 2.2. 구성 요소

1.  **모집 정보 섹션**:
    -   **그룹 정보**: 페이지 상단에 모집을 진행하는 그룹의 로고와 이름 표시. 클릭 시 해당 그룹의 홈으로 이동.
    -   **모집 제목 및 내용**: `recruitment-system.md`의 `title`, `content`에 해당하는 내용 표시.
    -   **핵심 정보**: 마감일, 모집 인원(설정 시), 지원 현황(공개 시) 등의 정보를 아이콘과 함께 요약하여 보여줍니다.

2.  **지원서 작성 섹션**:
    -   "지원하기" 버튼을 누르거나 페이지 하단으로 스크롤하면 나타납니다.
    -   "지원서 작성"이라는 제목 아래, `recruitment-system.md`의 `applicationQuestions`에 정의된 질문들이 `Text Field` (주로 `multiline`)로 나열됩니다.
    -   각 질문 필드 위에는 명확한 `Label`로 질문 내용이 표시됩니다.

### 2.3. 사용자 플로우

1.  사용자가 페이지 하단의 "지원하기" `Filled Button`을 클릭합니다.
2.  모든 필수 질문에 답변을 입력하면 "제출하기" 버튼이 활성화됩니다.
3.  "제출하기" 클릭 시, "지원서를 제출하시겠습니까?" `Dialog`를 통해 재확인합니다.
4.  확인 시 지원서가 제출되며, "지원이 완료되었습니다" `Snackbar` 메시지를 표시하고, 사용자는 `3. 지원 현황 확인 페이지`로 이동합니다.
5.  이미 지원한 경우, "지원하기" 버튼은 "내 지원서 확인하기"로 변경되며, 클릭 시 `3. 지원 현황 확인 페이지`로 바로 이동합니다.

## 3. 지원 현황 확인 페이지

사용자가 자신이 제출한 지원서의 상태를 확인하고 철회하는 페이지입니다.

### 3.1. 구성 요소

-   **페이지 제목**: "내 지원서"
-   **지원 상태**: 페이지 상단에 `Chip` 컴포넌트를 사용하여 현재 지원서의 상태(`APPLICATION_COMPLETE`, `APPROVED`, `DENIED`, `WITHDRAWN`)를 명확하게 표시합니다. (예: "신청 완료", "승인됨", "반려됨", "철회됨")
-   **제출 내용**: 사용자가 작성했던 지원서 내용을 수정 불가능한 `read-only` 형태로 보여줍니다.
-   **액션 버튼**:
    -   `APPLICATION_COMPLETE` 상태일 때만 "지원 철회하기" `Outlined Button`이 표시됩니다.
    -   클릭 시, "지원을 철회하면 재신청이 가능합니다. 정말 철회하시겠습니까?" `Dialog`로 확인 후 처리합니다.
-   **심사 의견 확인**:
    -   `DENIED` 상태일 때만 "반려 사유 확인" `Text Button`이 표시됩니다.
    -   클릭 시, 관리자가 작성한 심사 의견(반려 사유)을 보여주는 `Dialog`를 띄웁니다.

### 3.2. 삭제된 공고 처리

-   만약 지원했던 모집 공고가 삭제된 경우, 지원 현황 페이지의 공고 정보 영역에 '[삭제된 공고]'와 같이 표시되며, 해당 공고의 상세 페이지로는 이동할 수 없습니다.

## 4. 모집 공고 작성/수정 페이지 (`RECRUITMENT_MANAGE` 권한자용)

### 4.1. 구성 요소

#### 질문 입력 UI (지원서 질문 설정)
모집 공고 작성 시 지원자에게 물어볼 질문을 설정하는 섹션입니다.

**기술적 제약**:
- 최대 5개 질문 설정 가능
- 각 질문은 1,000자 이하

**UI 구조**:
```
📝 지원서 질문 설정
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
┌─────────────────────────────────┐
│ 질문 1                          │
│ [텍스트 입력 필드]              │
│                    [X 삭제]     │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ 질문 2                          │
│ [텍스트 입력 필드]              │
│                    [X 삭제]     │
└─────────────────────────────────┘

[➕ 질문 추가] (5개 미만일 때만 표시)
```

**동작**:
1. 초기 상태: 질문 입력 필드 1개 표시
2. "➕ 질문 추가" 버튼 클릭 → 새로운 입력 필드 추가 (최대 5개)
3. 5개 도달 시 "➕ 질문 추가" 버튼 비활성화
4. 각 질문 우측의 [X 삭제] 버튼으로 개별 삭제 가능
5. 최소 1개 질문은 유지 (마지막 질문은 삭제 불가)

**반응형 처리**:
- 데스크톱: 입력 필드 전체 너비, 삭제 버튼 우측 정렬
- 모바일: 입력 필드 축소, 삭제 버튼은 아이콘만 표시

## 5. 지원자 관리 페이지 (`RECRUITMENT_MANAGE` 권한자용)

모집 담당자가 지원자 목록을 보고 지원서를 심사하는 페이지입니다.

### 5.1. 디자인 컨셉

-   **효율적인 심사**: 지원자 목록을 테이블 또는 리스트 형태로 제공하여 상태별로 정렬 및 필터링할 수 있도록 합니다.
-   **신속한 결정**: 각 지원자 항목에서 즉시 '승인' 또는 '거절' 액션을 취할 수 있는 UI를 제공합니다.

### 5.2. 구성 요소

1.  **페이지 제목**: "지원자 관리"
2.  **상태 필터**: "전체", "신청 완료", "승인", "반려", "철회" 상태를 선택할 수 있는 `Tabs` 또는 `Segmented Button`을 제공하여 지원자 목록을 필터링합니다.
3.  **지원자 리스트**:
    -   각 지원자는 리스트의 한 항목으로 표시됩니다.
    -   **항목 내용**: 지원자 프로필 이미지, 닉네임, 지원 날짜, 현재 상태(`Chip` 활용).
    -   항목 클릭 시, 지원서 상세 내용을 볼 수 있는 `Dialog` 또는 별도 페이지로 이동합니다.
    -   **정렬 및 재지원 표시**: 철회된 지원서는 리스트의 가장 하단에 옅은 색상 등 비활성화된 형태로 표시하여 다른 지원서와 구분합니다. 한 지원자가 철회 후 재지원한 경우, 활성 지원서와 철회된 지원서가 모두 리스트에 각자의 상태로 표시됩니다.
4.  **심사 액션**:
    -   "신청 완료" 상태의 지원자 항목 우측에 "승인" `Filled Button`과 "반려" `Outlined Button`을 배치합니다.
    -   "반려" 클릭 시, 간단한 사유를 입력할 수 있는 `Dialog`를 띄웁니다.
    -   심사가 완료되면 해당 지원자의 상태가 변경되고 리스트가 업데이트됩니다.

### 5.3. 반응형 레이아웃 (멤버 관리 페이지 패턴)

#### 데스크톱 (768px 이상)
**테이블 레이아웃**:
```
[지원자 | 지원 날짜 | 상태 | 심사자 | 액션]
─────────────────────────────────────────
[Row 1] 데이터...
[Row 2] 데이터...
```
- `Expanded(flex)` 비율 기반 열 너비
- 흰색 배경, 둥근 모서리 (`AppRadius.card`), 회색 테두리 (`AppColors.neutral300`)
- 구분선 (`Divider`)으로 행 분리
- 우측에 승인/반려 버튼 배치

#### 모바일 (768px 미만)
**카드 레이아웃**:
```
┌─────────────────────────────────┐
│ [아바타] 지원자명               │
│ 지원일: 2025.10.10             │
│ 상태: [신청 완료] Chip         │
│ [승인] [반려] 버튼             │
└─────────────────────────────────┘
```
- 각 지원자를 독립된 카드로 표시
- 패딩 16px, 카드 간격 12px
- 아이콘 + 텍스트 조합
