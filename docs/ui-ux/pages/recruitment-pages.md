# 모집 페이지 (Recruitment Pages)

그룹 신규 멤버 모집 관련 전체 페이지의 개요와 네비게이션 허브입니다.

## 개요

사용자는 모집 공고를 확인하고 지원하며, 관리자는 지원자를 관리합니다. 사용자와 관리자를 위한 별도 페이지 세트로 구성됩니다.

## 페이지 구성

모집 시스템은 크게 두 가지 사용자 그룹으로 구분됩니다:

### 1. 사용자 대상 페이지
**문서**: [recruitment-user-pages.md](recruitment-user-pages.md)

일반 사용자가 모집 공고를 탐색하고 지원하는 페이지들:
- 모집 공고 리스트 페이지 (검색 기능, 카드 레이아웃)
- 모집 공고 상세 페이지 (지원서 작성)
- 지원 현황 확인 페이지 (지원서 상태 확인 및 철회)

### 2. 관리자 대상 페이지
**문서**: [recruitment-admin-pages.md](recruitment-admin-pages.md)

`RECRUITMENT_MANAGE` 권한을 가진 사용자가 사용하는 관리 페이지들:
- 모집 공고 작성/수정 페이지 (질문 입력 UI)
- 지원자 관리 페이지 (탭 구조, 상태 필터, 심사 액션)

## 페이지 네비게이션 구조

### 접근 경로

- **일반 사용자**: 홈 > 그룹 상세 > 모집 공고 (공개)
- **관리자**: 워크스페이스 > 관리자 메뉴 > 모집 관리

### 관리자 페이지 구조 (탭 기반)

관리자 메뉴의 "모집 관리" 버튼 클릭 시 다음과 같은 탭 구조로 표시:

```
[모집 공고 관리] [지원자 관리]
─────────────────────────────────
(현재 탭 콘텐츠)
```

**기술 구현**:
- `TabController` 사용 (멤버 관리 페이지와 동일한 패턴)
- 반응형 분기: `ResponsiveBreakpoints.of(context).largerThan(MOBILE)` (768px 기준)
- 데스크톱: 테이블 레이아웃
- 모바일: 카드 레이아웃

## 주요 기능

### 사용자
- 모집 공고 탐색 (검색/필터)
- 지원서 제출 (질문 답변)
- 지원 현황 확인 (상태, 철회)

### 관리자
- 모집 공고 작성/수정 (질문 최대 5개)
- 지원자 관리 (상태별 필터)
- 승인/반려 심사 (반려 사유 작성)

## 반응형
- **데스크톱** (768px+): 테이블 레이아웃
- **모바일**: 카드 레이아웃 (패딩 16px, 간격 12px)
