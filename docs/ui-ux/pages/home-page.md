# UI/UX 명세서: 홈 페이지

본 문서는 앱의 시작점인 홈 페이지의 구조와 기능을 정의합니다. 홈 페이지는 교내의 전반적인 소식을 접하고, 다양한 그룹과 활동을 '발견'하는 관문 역할을 수행합니다.

## 1. 디자인 철학

-   **발견의 즐거움**: 사용자가 미처 몰랐던 새로운 그룹이나 교내의 주요 행사를 쉽게 발견하고 참여할 수 있도록 돕습니다.
-   **핵심 정보 전달**: 복잡한 개인화 정보 대신, 모든 학생에게 해당될 수 있는 중요한 공지나 인기 있는 콘텐츠를 전면에 배치합니다.
-   **명확한 진입점**: 그룹 탐색, 모집 공고 확인 등 앱의 주요 탐색 기능으로 진입하는 명확한 입구를 제공합니다.

## 2. 화면 레이아웃

-   화면 상단에는 앱의 로고와 알림 아이콘을 배치합니다.
-   그 아래로 사용자가 스크롤하며 확인할 수 있는, 보다 공적인 성격의 위젯(카드)들이 수직으로 나열됩니다.

## 3. 구성 요소 (위젯)

### 3.1. 주요 학사 공지 (Key University Announcements)

-   **목적**: 학교 전체에 해당하는 중요 공지(예: 수강신청, 장학금 신청, 학사일정 변경)를 놓치지 않도록 전달합니다.
-   **표시 내용**:
    -   관리자가 등록한 주요 공지사항의 제목을 최신순으로 2~3개 보여주는 배너 또는 슬라이드 형태로 표시합니다.
-   **상호작용**: 클릭 시 공지사항 상세 내용 페이지로 이동합니다.

### 3.2. 지금 모집 중인 그룹 (Groups Recruiting Now)

-   **목적**: 사용자가 새로운 그룹을 탐색하고 합류할 기회를 쉽게 발견하도록 돕니다.
-   **표시 내용**:
    -   현재 모집 중인 그룹 중 2~3개를 `Card` 형태로 보여줍니다.
    -   각 카드는 [그룹 로고], [그룹명], [모집 제목], [마감 N일 전] 정보를 포함합니다.
-   **상호작용**:
    -   위젯 헤더의 '전체 보기'를 클릭하면 `모집 공고 리스트 페이지`로 이동합니다.
    -   개별 카드를 클릭하면 해당 `모집 공고 상세 페이지`로 이동합니다.

### 3.3. 요즘 인기 있는 그룹 (Popular Groups Lately)

-   **목적**: 최근 활동이 활발하거나 신규 멤버가 많이 가입한 그룹을 노출하여 사용자의 흥미를 유발하고 그룹 탐색을 돕니다.
-   **표시 내용**:
    -   "최근 일주일간 가장 많은 게시글이 작성된 그룹", "가장 많은 멤버가 가입한 그룹" 등의 기준으로 3개 정도의 그룹을 랭킹 형태로 보여줍니다.
    -   각 항목은 [순위], [그룹명], [간단한 소개]를 표시합니다.
-   **상호작용**: 그룹명을 클릭하면 해당 그룹의 공개 홈 페이지로 이동합니다.

### 3.4. 전체 그룹 둘러보기 (Explore All Groups)

-   **목적**: 사용자가 교내 모든 그룹의 구조를 탐색하고 새로운 관심 그룹을 발견할 수 있는 입구를 제공합니다.
-   **표시 내용**:
    -   "우리 학교의 모든 동아리, 학회, 학생회를 한눈에 둘러보세요!" 와 같은 행동 유도 문구가 담긴 넓은 배너 `Card`로 디자인합니다.
-   **상호작용**: 배너 클릭 시, 아래 4번 항목에서 설명하는 '전체 그룹 탐색 페이지'로 이동합니다.

## 4. 전체 그룹 탐색 페이지 상세 명세

홈 페이지의 '전체 그룹 둘러보기' 배너를 통해 진입하는 페이지의 상세 명세입니다.

### 4.1. 핵심 목표

-   **직관적인 탐색**: 사용자가 복잡한 그룹 구조를 마치 파일 탐색기처럼 쉽게 이해하고 원하는 그룹을 찾을 수 있도록 돕습니다.
-   **정보의 발견**: 특정 목적 없이 탐색하는 사용자도 흥미로운 그룹을 발견할 수 있도록 유도합니다.
-   **확장성**: 중앙동아리, 단과대학, 학과, 소모임 등 다양한 수준과 형태의 그룹을 일관된 방식으로 표현할 수 있는 구조를 제공합니다.

### 4.2. 화면 구조 및 기능

-   **페이지 제목**: "전체 그룹"
-   **검색 바**: 페이지 상단에 위치하며, 그룹 이름으로 즉시 검색할 수 있는 기능을 제공합니다.
-   **트리 뷰(Tree View)**: 페이지의 핵심 영역으로, 계층 구조로 그룹을 표시합니다.

### 4.3. 트리 뷰 동작 방식

-   **최상위 노드 (Root Nodes)**:
    -   **총학생회**: 총학생회 산하 기구들을 포함합니다.
    -   **중앙동아리**: 분과(예: 봉사, 학술, 종교, 체육, 공연)별로 그룹화된 중앙동아리 목록을 포함합니다.
    -   **단과대학**: 각 단과대학 학생회 및 산하 학과/학부 학생회를 계층적으로 보여줍니다.

-   **노드(Node) 구성**:
    -   각 노드는 `아이콘`, `그룹명`, `펼치기/접기 토글(> V)`로 구성됩니다.
    -   최하위 노드(실제 그룹)는 `그룹 아이콘`과 `그룹명`만 표시됩니다.
    -   노드 옆에는 대학그룹이라면 "대학", "단과대", "학과" 라고 쓰여진 칩이 있고, 속한 그룹이라면 "내 대학", "내 단과대", "내 학과"와 같은 칩이 추가로 표시됩니다.
    -   자율그룹에는 "자율 그룹", 공식 그룹에는 "공식 그룹"이라는 칩이 붙으며, 속한 그룹이라면 "내 그룹" 이라는 칩이 추가됩니다.

-   **상호작용**:
    -   **펼치기/접기**: 상위 노드(카테고리)의 토글 아이콘을 클릭하면 하위 노드 목록이 나타나거나 사라집니다. (애니메이션 효과 포함)
    -   **그룹 상세 보기**: 최하위 노드(그룹)를 클릭하면 해당 그룹의 '공개 홈 페이지'로 이동합니다. 사용자가 가입되어 있는 그룹이라면, 해당 그룹의 워크스페이스로, 가입이 되지 않았다면 그룹의 소개 페이지로 이동합니다.
    -   **기본 상태**: 페이지 진입 시 모든 노드는 접혀 있는 상태(Collapsed)로 시작하여 사용자가 원하는 부분부터 탐색하도록 유도합니다. 사용자가 가입되어 있는 그룹은 펼쳐져 있습니다.

### 4.4. 검색 기능

-   **동작**: 검색 바에 키워드를 입력하면, 해당 키워드를 포함하는 그룹명을 가진 노드만 필터링하여 보여줍니다.
-   **결과 표시**:
    -   검색 결과는 단순 리스트가 아닌, **원래의 트리 구조를 유지**한 채로 필터링된 결과만 보여줍니다.
    -   예를 들어, '농구'를 검색하면 '중앙동아리 > 체육분과 > **농구**동아리' 와 같이 상위 경로가 모두 펼쳐진 상태로 하이라이트되어 표시됩니다.
    -   이를 통해 사용자는 검색된 그룹이 어떤 소속인지 명확하게 인지할 수 있습니다.
