# UI/UX 명세서: 내 활동 페이지

본 문서는 사용자 개인과 직접적으로 관련된 모든 활동과 정보를 시간순으로 모아보는 '내 활동' 피드의 구조와 기능을 정의합니다. 이 페이지는 앱의 핵심 개인화 허브 역할을 합니다.

## 1. 디자인 철학

-   **통합된 타임라인**: 내가 참여하는 모든 그룹의 활동, 내 일정, 나에게 온 요청 등을 하나의 타임라인에 모아놓아, 사용자가 여러 화면을 방문할 필요 없이 이곳에서 모든 것을 파악할 수 있게 합니다.
-   **역할 중심 필터링**: 정보를 '나에게 온 알림'과 '내가 한 활동'으로 나누고, 그 안에서 세부 필터링을 제공하여 사용자가 원하는 정보만 선택적으로 볼 수 있게 합니다.
-   **빠른 상호작용**: 피드의 각 항목 내에서 '참석/불참', '지원서 검토', '투표하기' 등 다음 행동을 즉시 수행할 수 있도록 버튼과 링크를 제공합니다.

## 2. 화면 레이아웃

-   **상단 탭**: 화면 상단에 `Tabs` 컴포넌트를 사용하여 **[전체 피드]**, **[내가 할 일]**, **[나의 활동 기록]** 필터를 배치합니다.
-   **활동 피드**: 그 아래로 필터링된 활동 내역이 시간의 역순(최신순)으로 스크롤 가능한 리스트 형태로 표시됩니다.

## 3. 구성 요소 (필터별 피드 아이템)

### 3.1. [전체 피드]

-   **목적**: 나와 관련된 모든 수신 활동(일정, 새소식, 할 일)을 통합하여 보여주는 기본 뷰입니다.
-   **표시 내용**: 아래 `내가 할 일`과 `새소식/일정`의 모든 아이템이 시간순으로 섞여서 표시됩니다.

### 3.2. [내가 할 일] (To-Do)

-   **목적**: 사용자의 응답이나 행동이 필요한 모든 사항을 모아서 보여주는 '할 일 목록'입니다.
-   **표시 내용**:
    -   **지원서 검토 (관리자)**: "관리중인 **[그룹명]**에 새로운 지원서가 도착했습니다. (지원자: OOO)"
    -   **투표 참여**: "**[그룹명]**에서 새로운 투표를 시작했습니다: *투표 제목*"
    -   **일정 참석 여부 응답**: "**[그룹명]**의 '정기 회의' 일정에 대한 참석 여부를 알려주세요."
    -   **지원 결과 확인**: "지원하신 **[그룹명]**의 모집 결과가 나왔습니다."
-   **상호작용**:
    -   **지원서**: '검토하기' 버튼 클릭 시 `지원자 관리 페이지`로 이동합니다.
    -   **투표**: '투표하기' 버튼 클릭 시 즉시 투표 `Dialog`가 열립니다.
    -   **일정**: '참석/불참' 버튼을 통해 즉시 응답할 수 있습니다.

### 3.3. [나의 활동 기록] (My Activity Log)

-   **목적**: 사용자가 앱에서 수행한 주요 활동들을 시간순으로 기록하고, 쉽게 되돌아볼 수 있게 합니다.
-   **표시 내용**:
    -   **게시글/댓글 작성**: "**[그룹명]** '자유게시판'에 새 글을 작성했습니다."
    -   **일정 생성/참여**: "'스터디' 일정을 생성했습니다." / "'선배와의 대화'에 참여 신청했습니다."
    -   **그룹 가입/지원**: "**[AISC]** 그룹에 가입했습니다." / "**[B-Maker]** 그룹에 지원했습니다."
    -   **투표 참여**: "**[총학생회]**의 '축제 슬로건 투표'에 참여했습니다."
-   **상호작용**: 각 항목을 클릭하면 해당 활동의 결과물(내가 쓴 게시글, 참여한 투표 결과 등) 페이지로 이동합니다.
