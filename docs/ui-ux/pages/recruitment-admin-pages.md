# 모집 페이지: 관리자 대상 (Recruitment Admin Pages)

본 문서는 모집 담당자가 사용하는 모집 공고 작성/수정 및 지원자 관리 페이지의 UI/UX를 정의합니다.

## 개요

`RECRUITMENT_MANAGE` 권한을 가진 사용자는 이 페이지들을 통해 모집 공고를 작성/수정하고, 지원자를 심사합니다.

## 접근 경로

- **관리자**: 워크스페이스 > 관리자 메뉴 > 모집 관리

## 관리자 페이지 구조 (탭 기반)

관리자 메뉴의 "모집 관리" 버튼 클릭 시 다음과 같은 탭 구조로 표시:

```
[모집 공고 관리] [지원자 관리]
─────────────────────────────────
(현재 탭 콘텐츠)
```

**기술 구현**:
- `TabController` 사용 (멤버 관리 페이지와 동일한 패턴)
- 반응형 분기: `ResponsiveBreakpoints.of(context).largerThan(MOBILE)` (768px 기준)
- 데스크톱: 테이블 레이아웃
- 모바일: 카드 레이아웃

## 1. 모집 공고 작성/수정 페이지

### 1.1. 구성 요소

#### 질문 입력 UI (지원서 질문 설정)

모집 공고 작성 시 지원자에게 물어볼 질문을 설정하는 섹션입니다.

**기술적 제약**:
- 최대 5개 질문 설정 가능
- 각 질문은 1,000자 이하

**UI 구조**:
```
📝 지원서 질문 설정
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
┌─────────────────────────────────┐
│ 질문 1                          │
│ [텍스트 입력 필드]              │
│                    [X 삭제]     │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ 질문 2                          │
│ [텍스트 입력 필드]              │
│                    [X 삭제]     │
└─────────────────────────────────┘

[➕ 질문 추가] (5개 미만일 때만 표시)
```

**동작**:
1. 초기 상태: 질문 입력 필드 1개 표시
2. "➕ 질문 추가" 버튼 클릭 → 새로운 입력 필드 추가 (최대 5개)
3. 5개 도달 시 "➕ 질문 추가" 버튼 비활성화
4. 각 질문 우측의 [X 삭제] 버튼으로 개별 삭제 가능
5. 최소 1개 질문은 유지 (마지막 질문은 삭제 불가)

**반응형 처리**:
- 데스크톱: 입력 필드 전체 너비, 삭제 버튼 우측 정렬
- 모바일: 입력 필드 축소, 삭제 버튼은 아이콘만 표시

## 2. 지원자 관리 페이지

모집 담당자가 지원자 목록을 보고 지원서를 심사하는 페이지입니다.

### 2.1. 디자인 컨셉

-   **효율적인 심사**: 지원자 목록을 테이블 또는 리스트 형태로 제공하여 상태별로 정렬 및 필터링할 수 있도록 합니다.
-   **신속한 결정**: 각 지원자 항목에서 즉시 '승인' 또는 '거절' 액션을 취할 수 있는 UI를 제공합니다.

### 2.2. 구성 요소

1.  **페이지 제목**: "지원자 관리"
2.  **상태 필터**: "전체", "신청 완료", "승인", "반려", "철회" 상태를 선택할 수 있는 `Tabs` 또는 `Segmented Button`을 제공하여 지원자 목록을 필터링합니다.
3.  **지원자 리스트**:
    -   각 지원자는 리스트의 한 항목으로 표시됩니다.
    -   **항목 내용**: 지원자 프로필 이미지, 닉네임, 지원 날짜, 현재 상태(`Chip` 활용).
    -   항목 클릭 시, 지원서 상세 내용을 볼 수 있는 `Dialog` 또는 별도 페이지로 이동합니다.
    -   **정렬 및 재지원 표시**: 철회된 지원서는 리스트의 가장 하단에 옅은 색상 등 비활성화된 형태로 표시하여 다른 지원서와 구분합니다. 한 지원자가 철회 후 재지원한 경우, 활성 지원서와 철회된 지원서가 모두 리스트에 각자의 상태로 표시됩니다.
4.  **심사 액션**:
    -   "신청 완료" 상태의 지원자 항목 우측에 "승인" `Filled Button`과 "반려" `Outlined Button`을 배치합니다.
    -   "반려" 클릭 시, 간단한 사유를 입력할 수 있는 `Dialog`를 띄웁니다.
    -   심사가 완료되면 해당 지원자의 상태가 변경되고 리스트가 업데이트됩니다.

### 2.3. 반응형 레이아웃 (멤버 관리 페이지 패턴)

#### 데스크톱 (768px 이상)
**테이블 레이아웃**:
```
[지원자 | 지원 날짜 | 상태 | 심사자 | 액션]
─────────────────────────────────────────
[Row 1] 데이터...
[Row 2] 데이터...
```
- `Expanded(flex)` 비율 기반 열 너비
- 흰색 배경, 둥근 모서리 (`AppRadius.card`), 회색 테두리 (`AppColors.neutral300`)
- 구분선 (`Divider`)으로 행 분리
- 우측에 승인/반려 버튼 배치

#### 모바일 (768px 미만)
**카드 레이아웃**:
```
┌─────────────────────────────────┐
│ [아바타] 지원자명               │
│ 지원일: 2025.10.10             │
│ 상태: [신청 완료] Chip         │
│ [승인] [반려] 버튼             │
└─────────────────────────────────┘
```
- 각 지원자를 독립된 카드로 표시
- 패딩 16px, 카드 간격 12px
- 아이콘 + 텍스트 조합
