# 홍보/모집 UI·UX 명세서 (Toss 철학 적용 · Flutter/Spring Boot) v1.0

> 목적: 지원자는 **10초 내 원하는 공고 발견**, 그룹장은 **1분 내 공고 작성/마감**. 토스 철학(한 화면=한 목표 · 가치 먼저 · 에러는 다음으로 안내 · 일관성=토큰/컴포넌트)을 모든 화면/문구/상태에 반영.

---

## 0. Design Principles (요약)
- **한 화면 = 한 목표**: 목록=발견, 상세=이해/이동, 작성=등록, 관리=마감/수정으로 목표 단일화.
- **가치 먼저**: 상세 첫 화면에 지원 가치(D-day, 핵심 태그, 활동 요약) 노출 → 이후 비용(읽기/이동).
- **에러는 다음으로 안내**: 활성 공고 1개 제한, 기간 오류 등은 차단이 아니라 **다음 행동(수정/마감 이동)**을 제시.
- **일관성**: 배지/칩/토스트/모달/Empty/스켈레톤/정렬/검색 UI를 공용 컴포넌트로.
- **짧고 친근한 라이팅**: 해요체, 결과 선언형(“등록했어요”).

---

## 1. 범위 & KPI
- **범위(MVP)**: 모집 게시판(목록/검색) · 공고 작성/수정/삭제/마감 · 공고 상세 · 그룹 페이지 딥링크.
- **KPI**
  - 발견 속도: 목록 진입→상세 클릭 **p50 ≤ 10s**
  - 작성 속도: 작성 열기→등록 완료 **p50 ≤ 60s**
  - 오류율: 작성 차단/기간 오류/중복 활성 시도 **< 1%**
  - 전환: 상세→그룹 페이지 이동 CTR **≥ 35%**

---

## 2. 페르소나 & 권한
- **그룹장/스태프(모집 관리 권한)**: 공고 작성·수정·삭제·마감.
- **일반 사용자(지원자)**: 목록 조회, 상세 열람, 그룹 페이지로 이동 후 가입 신청.
- **시스템**: 종료일 자동 마감, 활성 1개 정책 강제.

---

## 3. 정보 구조 & 라우팅(IA)
- **탭: 모집** → **목록(모든 활성 공고)**
- **상세**: `/recruitments/:id` (태그/기간/D-day/본문/CTA)
- **작성/수정**: `/recruitments/new` · `/recruitments/:id/edit` (권한자 전용)
- **관리 숏컷**: 그룹 관리 > 모집(내 공고) → [수정] [마감] [삭제]
- **딥링크**: 상세 하단 [그룹 페이지로 이동] → 그룹 상세 → [가입 신청하기]

---

## 4. 상태 모델(State Model)
- **Active(모집 중)**: 목록/검색 노출, 상세 CTA 활성
- **Closed(마감)**: 목록 비노출, 상세 상단에 "마감됨" 배지 + CTA 비활성
- **Scheduled(예정)** *(시작일>오늘, 선택)*: 목록 비노출, 작성자만 관리 화면에서 확인
- **Deleted**: 접근 차단(404 대체 안내)
- **자동 전환**: 종료일 경과 → Active → **Closed**

상태 전이(텍스트): `Scheduled → Active → Closed → (삭제 가능)` / `Active → 수동 마감 → Closed`

---

## 5. 화면 설계 (Screens)
### A. 모집 게시판(목록)
- **목표**: 원하는 공고를 빠르게 찾기.
- **헤더**: 페이지 타이틀 "모집" · 검색창(제목/태그) · (선택) 정렬 드롭다운(기본: 최신순)
- **리스트 아이템(카드)**
  - 상단: **제목**(한 줄), **태그 칩**(최대 3), **D-day**(D‑3 / D‑day / 마감)
  - 하단: **기간**(YYYY.MM.DD~MM.DD), **모집 인원**, **그룹명**
  - 상호작용: 카드 탭→상세
- **빈 상태**: "진행 중인 모집이 없어요" [탐색 탭으로 가기]
- **로딩**: 스켈레톤 카드 5개

### B. 공고 작성/수정
- **목표**: 입력 최소화로 1분 내 등록.
- **필드 순서**: 제목 → 본문(멀티라인) → 모집 기간(시작/종료) → 모집 인원(텍스트) → 태그(칩)
- **보조 정보**: 입력 예시(placeholder) 제공
- **상단 경고(활성 1개 정책)**: 기존 활성 공고 존재 시 **차단 패널**
  - 제목: "현재 진행 중인 모집 공고가 있어요"
  - 본문: "새 글을 작성하려면 기존 공고를 **마감**하거나 **수정**해 주세요"
  - 버튼: [기존 공고 보기] [마감하러 가기]
- **CTA**: [등록] (필수값 충족 시 활성) / [임시저장] *(선택)*
- **검증**: 종료일<시작일 → 인라인 에러, 제목/본문 공백 금지, 태그 중복 제거
- **성공 피드백**: 토스트 "공고를 등록했어요" → 상세로 이동

### C. 공고 상세
- **헤더 블록**: 제목(H1) · 태그 칩 · **상태 배지(D‑n / 마감됨)**
- **메타 정보 바**: 기간, 모집 인원, 그룹명
- **본문**: 서식 지원(볼드/리스트) *(이미지 첨부는 차순위)*
- **CTA(스티키)**: [그룹 페이지로 이동] → 그룹 상세 → [가입 신청하기]
- **마감 상태**: 상단 배지 "마감됨" + CTA 비활성(툴팁 "마감된 공고예요")
- **작성자 퀵액션**: 상단 … 메뉴 [수정] [마감] [삭제]

### D. 관리(내 공고)
- **목표**: 상태 전환/수정을 빠르게.
- **리스트**: 내 공고 카드(상태/기간/제목) + [수정] [마감] [삭제]
- **마감 확인 모달**: "지금 마감할까요?" [마감] [취소]
- **삭제 확인 모달**: 영향 안내(되돌릴 수 없음)

## 7. 마이크로카피 (Toss 톤)
- 등록 성공: **"공고를 등록했어요"**
- 수정 성공: **"수정을 저장했어요"**
- 삭제 성공: **"공고를 삭제했어요"**
- 조기 마감: **"공고를 마감했어요"**
- 자동 마감: **"종료일이 지나 자동으로 마감되었어요"**
- 작성 차단: **"현재 진행 중인 모집 공고가 있어 새 글을 작성할 수 없어요"**
- 기간 오류: **"종료일은 시작일 이후여야 해요"**

## 8. 검증 규칙 & 예외 처리
- **필수값**: 제목(최소 1자), 본문(최소 1자), 기간(둘 다), 태그(≥1 권장)
- **기간**: `end >= start` 필수, `start ≤ 오늘` 허용(즉시 게시). `start > 오늘`이면 Scheduled 처리
- **활성 공고 1개 제한**: 동일 그룹 Active가 존재하면 작성 화면 **진입 차단** + PolicyBlock 표시
- **마감 처리**: 종료일 경과 시 자동 Closed + 목록 비노출, 상세에 배지 표기
- **삭제**: 세컨더리 확인 모달 + 되돌릴 수 없음 안내

## 11. API 연동(UX 관점)
- **GET /recruitments?query=&tags=** → 활성 공고 목록(페이징)
- **GET /recruitments/{id}** → 상세
- **POST /recruitments** `{title, body, startDate, endDate, capacityText, tags[]}`
- **PATCH /recruitments/{id}** (수정)
- **POST /recruitments/{id}/close** (수동 마감)
- **DELETE /recruitments/{id}** (삭제)
- **GET /groups/{groupId}/recruitments/mine** (내 공고 관리)
- **자동 마감 잡**: 매일 00:05 종료일 경과 항목 Closed 전환

## 13. 수용 기준(AC)
1) 활성 공고 존재 시 **작성 진입 차단** + 차단 패널 노출(두 액션 제공).
2) 종료일 경과 시 **자동 마감**되고, 목록에서 즉시 비노출.
3) 상세에서 [그룹 페이지로 이동] 버튼은 항상 하단 스티키로 노출.
4) 작성 성공 시 **상세 화면으로 이동**하며 2.5s 토스트가 노출.
5) 목록 로딩은 스켈레톤 ≥3개, 빈 상태 메시지/대체 행동 제공.
6) 기간 오류/필수값 누락은 **인라인 에러**로 즉시 안내.
