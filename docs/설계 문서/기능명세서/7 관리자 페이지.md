### **관리자 페이지 기능 명세서 (v2.0 - 설계 확정)**

> **🔵 구현 상태:** 설계 확정 (백엔드 구현 필요)
> **📅 최종 업데이트:** 2025-09-19

**1. 관리 홈 및 접근 정책**

- **접근 방식:** 하나 이상의 관리 권한을 가진 사용자에게만 '관리' 버튼이 노출되며, 이를 통해 '관리 홈'으로 진입합니다.
- **UI 컨셉:**
    - **카드/리스트 형태 메뉴:** 아이콘과 설명을 포함하여 관리 메뉴를 제공합니다.
    - **맥락적 정보 제공:** `멤버 관리` 메뉴 옆에 `가입 대기 N명` 뱃지를 표시하는 등, 다음에 할 일을 바로 인지하게 합니다.
    - **권한 기반 메뉴:** 사용자에게 권한이 있는 메뉴만 목록에 표시합니다.

**2. MVP 기능 목록**

- `멤버 관리`
- `역할 관리`
- `채널 관리`
- `지도교수 관리` (그룹장에게만 노출)
- `그룹 정보 수정`
- `그룹 삭제`

**3. 핵심 기능별 상세 흐름**

- **역할 및 채널 관리:** 여러 단계의 복잡한 흐름 대신, 한 페이지에서 모든 설정을 보고 저장하는 **'한 화면 설정'** 방식을 도입합니다.
- **그룹 삭제:** 삭제 시 **바텀 시트(Bottom Sheet)**를 통해 삭제될 데이터 정보를 명확히 안내하고, 사용자가 **그룹 이름을 직접 입력**해야만 활성화되는 강력한 확인 절차를 거칩니다.

### 4. API 명세 (설계 확정안)

| 기능 | HTTP Method | URI | 설명 |
| --- | --- | --- | --- |
| **관리 홈 요약 조회** | `GET` | `/api/groups/{groupId}/admin/stats` | 관리 홈에 필요한 통계(가입 대기 인원 등)를 조회합니다. |
| **멤버 관리 API** | (다수) | `/api/groups/{groupId}/members...` | '권한/멤버 관리' 명세서의 API를 재사용합니다. |
| **역할 관리 API** | (다수) | `/api/groups/{groupId}/roles...` | '권한/멤버 관리' 명세서의 API를 재사용합니다. |
| **채널 관리 API** | (다수) | `/api/groups/{groupId}/channels...` | '그룹 워크스페이스' 명세서의 API를 재사용합니다. |
| **지도교수 목록 조회** | `GET` | `/api/groups/{groupId}/professors` | 현재 그룹의 지도교수 목록을 조회합니다. |
| **지도교수 추가** | `POST` | `/api/groups/{groupId}/professors/{professorId}` | ID로 교수를 찾아 지도교수로 지정합니다. |
| **지도교수 삭제** | `DELETE` | `/api/groups/{groupId}/professors/{professorId}` | 지도교수를 그룹에서 제외합니다. |
| **그룹 정보 수정** | `PATCH` | `/api/groups/{groupId}` | 그룹의 이름, 설명 등 일부 정보를 수정합니다.<br>**(📝 개발 TODO: 현재 코드는 `PUT`으로 되어있으나, `PATCH`가 더 올바른 설계이므로 코드 수정 필요)** |
| **그룹 삭제** | `DELETE` | `/api/groups/{groupId}` | 그룹을 삭제합니다. (소프트 삭제) |

### 5. DTO (Data Transfer Object) 정의

- **AdminStatsDto**: `{ pendingMemberCount, totalMemberCount, roleCount, channelCount }`
- **UpdateGroupRequest (`PATCH`용)**: `{ name: String?, description: String?, tags: List<String>? }` (수정할 필드만 전송)
- **UserDto**: `{ userId, nickname, email, profileImageUrl }`
