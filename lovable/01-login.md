# 기능 명세: 로그인

## 1. 목표
사용자는 이메일과 비밀번호 또는 소셜 계정(구글)을 사용하여 서비스에 안전하게 인증하고 진입할 수 있다.

## 2. 화면 구성 및 기능 (구현 코드 기반)

### 2.1. UI 요소
- **로고:** 화면 상단에 서비스 로고가 표시된다.
- **이메일 입력 필드:** 사용자가 이메일 주소를 입력하는 텍스트 필드.
- **비밀번호 입력 필드:** 사용자가 비밀번호를 입력하는 텍스트 필드. (입력 내용 마스킹 처리)
- **로그인 버튼:** 이메일과 비밀번호 입력 후 클릭하여 로그인을 시도하는 버튼.
- **소셜 로그인 버튼:**
    - **"Sign in with Google" 버튼:** 클릭 시 구글 OAuth 인증 절차를 시작한다.
- **기타 링크:**
    - **회원가입:** 회원가입 페이지로 이동하는 텍스트 링크.
    - **비밀번호 찾기:** 비밀번호 재설정 페이지로 이동하는 텍스트 링크.

### 2.2. 사용자 인터랙션 및 동작
- **입력:** 사용자가 각 필드에 텍스트를 입력할 수 있다.
- **로그인 시도:**
    - 사용자가 '로그인' 버튼을 클릭하면, 입력된 이메일과 비밀번호의 유효성을 검증한다.
    - 유효성 검증 통과 시, 백엔드 API로 로그인 요청을 전송하며 로딩 인디케이터를 표시한다.
- **로그인 성공:**
    - API로부터 성공 응답(JWT 토큰 등)을 받으면, 인증 정보를 안전하게 저장한다.
    - 사용자를 서비스의 메인 화면(홈)으로 리디렉션한다.
- **로그인 실패:**
    - 잘못된 자격 증명, 서버 오류 등의 사유로 실패 응답을 받으면, 사용자에게 "이메일 또는 비밀번호를 확인해주세요."와 같은 적절한 에러 메시지를 스낵바 또는 토스트 형태로 보여준다.
- **구글 로그인:**
    - 'Sign in with Google' 버튼 클릭 시, 구글 인증 화면으로 전환된다.
    - 사용자가 구글 계정 선택 및 인증을 완료하면, 콜백을 통해 받은 인증 정보로 백엔드에 로그인을 요청한다.
    - 이후 프로세스는 일반 로그인 성공/실패와 동일하다.
