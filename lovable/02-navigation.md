# 기능 명세: 네비게이션 (상단바 및 하단바)

## 1. 목표
사용자는 앱의 주요 기능들 사이를 직관적으로 이동할 수 있으며, 현재 자신의 위치와 상태(역할, 알림 등)를 명확하게 인지할 수 있다.

## 2. 화면 구성 및 기능 (구현 코드 및 UI/UX 명세 기반)

### 2.1. 상단바 (App Bar)
- **역할:** 현재 화면의 컨텍스트를 알려주고, 알림 및 프로필 관련 빠른 접근을 제공한다.
- **구성 요소:**
    - **화면 제목:** 현재 보고 있는 페이지의 제목이 표시된다. (예: "채널", "공지", "멤버 관리")
    - **(워크스페이스 컨텍스트) 그룹명 및 역할 배지:** 사용자가 특정 워크스페이스 내부에 있을 경우, 현재 소속된 그룹의 이름과 해당 그룹에서의 사용자 역할(예: '그룹장', '멤버')을 배지로 표시한다.
    - **관리 버튼:** 사용자가 현재 워크스페이스에 대한 관리 권한(그룹장, 스태프 등)을 가지고 있을 경우에만 '관리' 버튼이 노출된다. 클릭 시 그룹 관리 페이지로 이동한다.
    - **알림 아이콘:** 클릭 시 알림 목록을 볼 수 있는 화면이나 드로어(drawer)를 연다. 읽지 않은 새 알림이 있을 경우, 아이콘 위에 붉은 점(badge)이 표시된다.

### 2.2. 하단 네비게이션 바 (Bottom Navigation Bar)
- **역할:** 앱의 최상위 핵심 기능들 사이를 전환하는 주된 수단이다. (모바일 뷰 기준)
- **구성 요소:**
    - **탭(Tabs):** 다음과 같은 주요 기능 탭으로 구성된다.
        - **홈:** 사용자가 속한 워크스페이스 목록 등 개인화된 메인 화면.
        - **탐색:** 새로운 그룹을 찾거나 탐색하는 화면.
        - **모집:** 그룹들이 게시한 모집 공고를 모아보는 화면.
        - **내 정보:** 사용자 프로필, 설정 등 개인 계정 관련 메뉴.
    - **동작:**
        - 각 탭 아이콘을 클릭하면 화면의 메인 콘텐츠 영역이 해당 기능 페이지로 즉시 전환된다.
        - 현재 활성화된 탭은 시각적으로 하이라이트 처리(예: 다른 색상, 굵은 아이콘)하여 사용자가 현재 위치를 명확히 알 수 있게 한다.
        - 화면 전환 시 이전 탭의 상태(예: 스크롤 위치)는 유지되어야 한다.

### 2.3. 반응형 동작 (데스크톱 뷰)
- 화면 너비가 900px 이상인 데스크톱 환경에서는 하단 네비게이션 바가 사라지고, 좌측에 고정된 사이드바 형태로 네비게이션이 제공된다.
- **Global Sidebar (전역 사이드바):** 하단 네비게이션 바의 기능들이 아이콘 형태로 최좌측에 고정된다.
- **Workspace Sidebar (워크스페이스 사이드바):** 사용자가 특정 워크스페이스에 진입했을 때, 그 옆에 채널 목록, 관리 메뉴 등이 포함된 두 번째 사이드바가 나타난다.
